export const covarianceEigenCode = `import numpy as np

# Generate 2D data with a clear elongated shape
np.random.seed(0)
n = 300
# Elongated cloud oriented at ~45 degrees
angle = np.pi / 4
R = np.array([[np.cos(angle), -np.sin(angle)],
              [np.sin(angle),  np.cos(angle)]])
S = np.diag([3.0, 0.7])                          # scale in x, then in y
X = np.random.randn(n, 2) @ S.T @ R.T             # rotate + scale a standard cloud

# Center the data (PCA REQUIRES centering)
X_centered = X - X.mean(axis=0)

# Sample covariance matrix (2x2, symmetric PSD)
C = np.cov(X_centered.T)                          # shape (2, 2)

# Eigen-decomposition of the covariance
eigvals, eigvecs = np.linalg.eigh(C)               # SORTED ASCENDING by eigh
order = np.argsort(-eigvals)                       # sort descending
eigvals = eigvals[order]
eigvecs = eigvecs[:, order]

print("Sample covariance C:")
print(np.round(C, 3))
print()
print("Eigenvalues (variance along each principal direction):", np.round(eigvals, 3))
print()
print("Eigenvectors (principal directions, columns):")
print(np.round(eigvecs, 3))
print()
print("First eigenvector points along the DATA'S long axis")
print("Second is perpendicular to it (spectral theorem — orthogonal)")
`;

export const pcaSVDCode = `import numpy as np

# Same 2D elongated cloud as above
np.random.seed(0)
n = 300
angle = np.pi / 4
R = np.array([[np.cos(angle), -np.sin(angle)],
              [np.sin(angle),  np.cos(angle)]])
S = np.diag([3.0, 0.7])
X = np.random.randn(n, 2) @ S.T @ R.T

# Center
X_centered = X - X.mean(axis=0)

# SVD of the CENTERED data matrix (n x d)
# X = U diag(singular_values) V.T
# The rows of V.T are the principal components (= eigenvectors of covariance)
U, sing_vals, Vt = np.linalg.svd(X_centered, full_matrices=False)

# Principal components:  each row of Vt is a component in feature space
components = Vt                                    # d x d
# Variances along principal directions (link between SVD and eigen):
# eigenvalues of covariance == (singular_values ** 2) / (n - 1)
variances = (sing_vals ** 2) / (n - 1)

print("Principal components (rows of Vt):")
print(np.round(components, 3))
print()
print("Variance explained by each component:", np.round(variances, 3))
print()
print("This matches the eigenvalue / eigenvector output of the previous cell.")
print()
print("Explained variance ratio (relative):",
      np.round(variances / variances.sum(), 3))
`;

export const pcaReduceCode = `import numpy as np

# Higher-dim data example: reduce 8-D data down to 2-D for visualization
np.random.seed(42)
n = 400
# True signal is 2-D, then embedded into 8-D by a random projection + noise
signal = np.random.randn(n, 2) * np.array([4.0, 1.0])   # anisotropic
proj = np.random.randn(2, 8) * 0.5
X = signal @ proj + 0.1 * np.random.randn(n, 8)          # 8-D observations

# 1. Center
Xc = X - X.mean(axis=0)

# 2. SVD
U, S, Vt = np.linalg.svd(Xc, full_matrices=False)

# 3. Sum explained variance and choose k
var = (S ** 2) / (n - 1)
cum = np.cumsum(var) / var.sum()
print("Cumulative explained variance:", np.round(cum, 3))

# 4. Project onto top-k principal directions
k = 2
X_reduced = Xc @ Vt[:k].T                                # n x k

print()
print(f"Reduced shape: {X_reduced.shape}")
print(f"Kept {100*cum[k-1]:.1f}% of variance with {k}/8 components")

# 5. Approximate reconstruction from k components (best rank-k under Frobenius norm)
X_approx = X_reduced @ Vt[:k]                            # n x d, back in feature space
recon_err = np.linalg.norm(Xc - X_approx) / np.linalg.norm(Xc)
print(f"Relative reconstruction error: {recon_err:.4f}")
`;

<Motivation title="Compressing a 1,024-dim embedding to 32 dimensions while keeping 95% of the signal">

Every embedding-based system in modern ML has the same shape: represent an object (a word, an image, a user, a molecule) as a d-dimensional vector, where d is 512, 1024, 1536, or more. Storage and compute grow with d. Downstream models train faster if d is small. Interpretability improves if you can compress to 2 or 3 dimensions for a plot.

But most of that d-dimensional space is *wasted*. Real embedding matrices have a small number of directions that account for most of the variance in the data; the rest is either noise or nearly-constant. If you find the right d′ &lt; d directions and project onto them, you keep most of the signal at a fraction of the cost.

That's PCA. Principal Component Analysis is the *canonical* algorithm for finding those directions — and, remarkably, it's just linear algebra: an eigendecomposition of the covariance matrix (or equivalently, an SVD of the centered data matrix). No optimization loop, no hyperparameters beyond "how many components to keep." Once you've built the intuition for eigenvectors and SVD, PCA falls out in one page.

This chapter brings everything from Phase 1.2 together — vectors, matrices, eigenvectors, SVD, positive-definite covariances — into one workflow: how to compress data while preserving what matters.

</Motivation>

<Intuition title="Find the directions of biggest variance, keep those, throw away the rest">

Imagine a cloud of 2D points shaped like an elongated ellipse. Any 1D summary of the cloud is a projection onto some line — but which line? Obviously the long axis of the ellipse: projecting onto that axis preserves most of the *spread* of the data, while projecting onto the short axis crushes almost all the differences between points into a narrow window.

**PCA generalizes that idea to n dimensions.** For a d-dimensional dataset, find the direction v₁ that captures the most variance when the data is projected onto it. Then find v₂, perpendicular to v₁, that captures the most of the remaining variance. And v₃ perpendicular to both, and so on. These vectors v₁, v₂, … are the **principal components**, and it turns out they're exactly the *eigenvectors of the data's covariance matrix*, ordered by decreasing eigenvalue.

Once you have them, "compress to k dimensions" is just "project onto the first k principal components." You keep the k directions of maximum variance and discard the d − k directions of noise or redundancy.

**One picture, three views of the same thing:**

- **Statistical**: the direction that maximizes variance of the projected data.
- **Geometric**: the long axis of the ellipsoid that best fits the data.
- **Linear algebra**: the top eigenvector of the covariance matrix (or the top-right singular vector of the centered data matrix).

The magic is that all three views give the same answer, and the algorithm to find them is just one eigen-decomposition or one SVD.

</Intuition>

<Recall term="Covariance matrix">

For centered data X (n samples, d features, mean subtracted), the sample covariance matrix is `Σ = (1/(n-1)) · Xᵀ X`. It's a d×d symmetric PSD matrix; Σ(i, j) is the covariance between feature i and feature j. Diagonal entries are variances of individual features.

</Recall>

## Route 1 — PCA as an eigen-problem on the covariance

Given a data matrix X of shape (n, d):

1. **Center**: subtract the column mean from each column, getting X_c. This is not optional — PCA is a statement about variance, and variance is defined relative to the mean.
2. **Form the covariance**: C = (1 / (n − 1)) · X_cᵀ X_c. This is a d×d symmetric PSD matrix.
3. **Eigen-decompose** C: `eigenvalues, eigenvectors = np.linalg.eigh(C)`. Sort in descending order of eigenvalue.

The top-k eigenvectors are the top-k principal components; the corresponding eigenvalues are the variances of the data projected onto each component. To reduce X to k dimensions:

$$
X_{\text{reduced}} \;=\; X_c \cdot V_k
$$

where V_k is the d×k matrix whose columns are the top-k eigenvectors.

<RunnableCode title="PCA via covariance eigendecomposition on a 2D elongated cloud" code={covarianceEigenCode} packages={["numpy"]} />

Notice how the first principal component aligns with the data's long axis (~45°) and captures ~4× the variance of the second component. The 2×2 covariance matrix encodes both the *spread* and the *orientation* of the cloud in one object — and eigendecomposition unpacks it.

## Route 2 — PCA via SVD of the centered data matrix

The eigendecomposition route works but has a numerical cost: forming the covariance matrix Xᵀ X squares the condition number, so tiny numerical errors in X become larger errors in C. The **SVD route** avoids this entirely.

If the SVD of the centered data matrix is:

$$
X_c \;=\; U\, \Sigma\, V^{\top}
$$

then:

- **The rows of Vᵀ (equivalently, columns of V) are the principal components.**
- **The variance along the k-th principal component is σ²_k / (n − 1)**, where σ_k is the k-th singular value.
- **Projecting X onto the top-k principal components** gives `X_reduced = U_k · Σ_k` (a shortcut — you can skip the explicit V multiplication).

The relationship to Route 1: eigenvectors of `C = Xᵀ X / (n − 1)` are exactly the right singular vectors of X_c, and eigenvalues of C are exactly the squared singular values divided by (n − 1). Same answer, better numerics.

<RunnableCode title="Same PCA via SVD of centered X — same answer, better conditioning" code={pcaSVDCode} packages={["numpy"]} />

**Rule of thumb**: for n or d small (say, up to 1000), either route is fine. For large n or d, always use SVD (via `np.linalg.svd` or, for very large data, truncated SVD with `sklearn.decomposition.TruncatedSVD` or `scipy.sparse.linalg.svds`). Never form the explicit covariance when d is large.

## The full PCA workflow

Combining everything into one pipeline that dimension-reduces a d-dimensional dataset to k dimensions:

1. **Center**: subtract column mean. (Optionally *standardize* by dividing by column std if features are on wildly different scales — but only if that makes sense for your problem.)
2. **Choose k** (see next section for the scree-plot approach).
3. **SVD**: `U, S, Vt = np.linalg.svd(X_centered, full_matrices=False)`. The top-k rows of `Vt` (equivalently, top-k columns of `V`) are the principal components.
4. **Project**: `X_reduced = X_centered @ Vt[:k].T`, shape (n, k).
5. **(Optional) reconstruct back**: `X_approx = X_reduced @ Vt[:k]`, shape (n, d). This is the best rank-k approximation to X_centered in Frobenius norm (Eckart–Young).
6. **Uncenter** for reconstruction: add back the mean.

## Choosing k — the scree plot

How many principal components should you keep? Two standard tools:

- **Cumulative explained variance**: plot `cumsum(σ²_i) / sum(σ²_j)` vs number of components. Choose the smallest k that captures, say, 90% or 95% of the total variance. This is the "keep enough to preserve the signal" heuristic.
- **Scree plot**: bar plot of variance-per-component. Look for an **"elbow"** — a point after which additional components contribute very little. The elbow is often where the "signal" ends and the "noise floor" begins.

<RunnableCode title="Scree plot + cumulative variance curve for a synthetic dataset" code={screePlotCode} packages={["numpy", "matplotlib"]} />

For images and text embeddings you often see a very clean elbow at 20–100 components; the tail is noise. For "wide" data (d large, n small), the meaningful number is at most n − 1 (a rank-nullity argument on X_centered — it can never have more than min(n − 1, d) nonzero singular values).

## What PCA is not, and its two biggest limitations

PCA is a **linear**, **variance-based**, **global** compressor. This gives it three limitations you should know:

- **Linear**: PCA can only capture axis-aligned linear structure (rotate, scale, project). It won't find curved manifolds (a swiss roll unrolled), interaction effects between features, or anything nonlinear. For nonlinear structure you need t-SNE, UMAP, or a nonlinear autoencoder.
- **Variance-based**: PCA gives priority to *high-variance* directions, which may or may not be the *discriminative* directions for a downstream task. A tiny-variance direction can be the most predictive of the label (imagine class means very close together and class variance small — PCA discards this signal). If you care about labels, use Linear Discriminant Analysis (LDA), or simply supervise your dim-reduction.
- **Global**: PCA gives one linear projection valid everywhere in the data. It cannot represent local structure — clusters that live in different subspaces of the same ambient space get mashed together.

For most preprocessing purposes (compress embeddings, initialize algorithms, denoise, visualize) PCA is the right first tool. When you need more, you know why to reach elsewhere.

<RealWorldML title="Embedding-model dimension reduction — cutting the 1,536-D OpenAI embedding to 128-D while keeping 99% of quality">

Modern text-embedding models (OpenAI's `text-embedding-3-large`, Cohere's models, sentence-transformers) return vectors of dimension 1,024 to 3,072. For a database of a million documents, storing at full dimension is ~6 GB per million; at 128-D it's ~500 MB. Query time (nearest-neighbor search) scales roughly linearly in dimension, so reducing to 1/12 the dimension makes queries 12× faster.

The empirical finding, reported in the OpenAI embeddings model card and confirmed by many teams: PCA-reducing these embeddings to 256 or 128 dimensions preserves *almost all* the retrieval quality. Loss of nDCG or accuracy is often less than 1% — while the storage and query cost drop by 10× or more.

Read the output:

- The synthetic embeddings were built to have intrinsic rank 16 (with noise). SVD discovers this: only ~16–20 components are needed to capture 95% of the variance.
- The compression ratio is roughly `d / (k*(n + 1 + d) / (n * d))` — for realistic (n, d) it's dominated by the ratio d/k. A 1,536-D embedding compressed to 128-D is 12× smaller.
- The reconstruction error stays tiny (a few percent) because the low-rank hypothesis holds — the embeddings *really are* nearly in a 16-D subspace, and everything past that is noise or negligible variation.

In production, the workflow is: (1) compute embeddings for a representative sample of your corpus with the full model, (2) fit PCA (or truncated SVD) once to learn the projection matrix, (3) apply the projection to every new embedding at ingestion or query time — one matrix multiply. The projection matrix is small (d × k) and cheap; you save the projection along with your embedding index. Cosine similarity in the reduced space closely tracks cosine similarity in the original space, so downstream retrieval is unchanged (up to the ~1% quality loss).

The connection to LoRA is direct: both are low-rank approximations, both exploit the fact that ML-relevant matrices are almost never full-rank in practice, and both trade a small amount of quality for a huge amount of efficiency. Once you see "the signal is intrinsically low-rank," rank-based tricks become the first thing to try in every compute-constrained ML system.

</RealWorldML>

<Summary>

- **PCA finds the directions of maximum variance** in a dataset. These directions — the *principal components* — are exactly the top eigenvectors of the data's covariance matrix, ordered by eigenvalue.
- **Two equivalent routes** to compute them: (a) eigendecomposition of `Xᵀ X / (n − 1)`, (b) SVD of the centered data matrix `X_c = U Σ Vᵀ` — the right singular vectors (rows of Vᵀ) are the principal components, and singular values give the variances via σ² / (n − 1).
- **Always center** before PCA (subtract column mean). Optionally *standardize* if features have wildly different scales. Never forget these preprocessing steps — they change the answer.
- **Prefer SVD** in code — `np.linalg.svd(X_centered)`. It's numerically more stable than forming the covariance and taking eigenvalues (which squares the condition number).
- **Choose k** with a *scree plot* (look for the elbow) or a *cumulative variance* threshold (e.g., keep 90% or 95%).
- **PCA gives the best rank-k linear approximation** to the data in Frobenius norm (Eckart–Young). This is the algebraic reason PCA-based compression works.
- **Limitations**: linear (can't capture curved manifolds), variance-based (ignores label information), global (one projection for all data). For those cases: t-SNE, UMAP, autoencoders, LDA.
- **In modern ML**: PCA is the go-to for compressing embeddings (10× storage savings at &lt;1% quality loss), initializing algorithms, denoising, and visualizing high-dim data. Same low-rank story as LoRA and truncated SVD.

</Summary>
