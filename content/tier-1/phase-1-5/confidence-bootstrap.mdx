export const frequentistCiCode = `# Frequentist CI: for large n, X_bar +- z * (s / sqrt(n)) with z from N(0,1).
# For small n, use t-distribution's quantile instead.
import numpy as np
from scipy.stats import norm, t as t_dist
rng = np.random.default_rng(0)

true_mean = 5.0
true_sigma = 2.0

# One realization: sample of size 30.
n = 30
sample = rng.normal(true_mean, true_sigma, size=n)
sample_mean = sample.mean()
sample_std = sample.std(ddof=1)     # unbiased (N-1 denominator)
se = sample_std / np.sqrt(n)

# CI using normal (large n).
z95 = norm.ppf(0.975)
ci_normal_low  = sample_mean - z95 * se
ci_normal_high = sample_mean + z95 * se

# CI using Student's t (correct for small n; converges to normal for n large).
t95 = t_dist.ppf(0.975, df=n-1)
ci_t_low  = sample_mean - t95 * se
ci_t_high = sample_mean + t95 * se

print(f"Sample: mean = {sample_mean:.4f}, std = {sample_std:.4f}, SE = {se:.4f}")
print(f"z-95%   (large-n approx): ({ci_normal_low:.4f}, {ci_normal_high:.4f})")
print(f"t-95%   (small-n exact):  ({ci_t_low:.4f}, {ci_t_high:.4f})")
print(f"Multiplier: z = {z95:.4f}, t (df=29) = {t95:.4f}   (t is slightly bigger for finite df)")

# Verify coverage: repeat the whole thing 5000 times and count what fraction of the
# CIs contain the true mean.
n_exp = 5000
hits_z, hits_t = 0, 0
for _ in range(n_exp):
    s = rng.normal(true_mean, true_sigma, size=n)
    m, sd = s.mean(), s.std(ddof=1)
    lo_z = m - z95 * sd / np.sqrt(n)
    hi_z = m + z95 * sd / np.sqrt(n)
    lo_t = m - t95 * sd / np.sqrt(n)
    hi_t = m + t95 * sd / np.sqrt(n)
    if lo_z <= true_mean <= hi_z: hits_z += 1
    if lo_t <= true_mean <= hi_t: hits_t += 1
print(f"\\nEmpirical coverage over {n_exp} samples of size {n}:")
print(f"  z-based 95% CI: {hits_z / n_exp * 100:.2f}%   (should be ~95%)")
print(f"  t-based 95% CI: {hits_t / n_exp * 100:.2f}%   (should be ~95%)")`;

export const bootstrapCode = `# The bootstrap: resample WITH REPLACEMENT from your original sample many times.
# The variability of your statistic across resamples estimates its sampling distribution.
import numpy as np
rng = np.random.default_rng(0)

# Original sample.  Use a skewed distribution so the sample mean's distribution
# isn't obviously Gaussian for small n.
true_mean = 2.0
n = 40
sample = rng.exponential(scale=2.0, size=n)   # E[X] = 2

print(f"Original sample: n = {n}, mean = {sample.mean():.4f}, std = {sample.std(ddof=1):.4f}")

# Bootstrap: draw n_boot resamples of size n WITH REPLACEMENT, compute the statistic on each.
n_boot = 5000
boot_means = np.zeros(n_boot)
for b in range(n_boot):
    idx = rng.integers(0, n, size=n)      # sample n indices with replacement
    boot_means[b] = sample[idx].mean()

# The bootstrap distribution of the sample mean.
print(f"\\nBootstrap distribution over {n_boot} resamples:")
print(f"  mean of bootstrap means: {boot_means.mean():.4f}    (approx original mean)")
print(f"  std  of bootstrap means: {boot_means.std():.4f}     (approx SE of the mean)")
print(f"  Compare to parametric SE = s/sqrt(n): {sample.std(ddof=1) / np.sqrt(n):.4f}")

# 95% CI via percentile method: take the 2.5 and 97.5 percentiles of the bootstrap distribution.
ci_low, ci_high = np.percentile(boot_means, [2.5, 97.5])
print(f"\\n95% percentile CI: ({ci_low:.4f}, {ci_high:.4f})")
print(f"True mean: {true_mean}  -- inside interval? {ci_low <= true_mean <= ci_high}")

# 95% CI via pivot/basic method: 2 * xbar - percentiles of boot means (opposite ends).
xbar = sample.mean()
pivot_low  = 2 * xbar - np.percentile(boot_means, 97.5)
pivot_high = 2 * xbar - np.percentile(boot_means,  2.5)
print(f"\\n95% pivot CI:     ({pivot_low:.4f}, {pivot_high:.4f})")
print("Two methods agree for symmetric bootstrap distributions; may differ for skewed ones.")`;

export const credibleIntervalCode = `# Bayesian credible interval: an interval [a, b] such that P(theta in [a, b] | data) = 0.95.
# This IS a probability about the parameter -- something a frequentist CI cannot say.
import numpy as np
from scipy.stats import beta as beta_dist
rng = np.random.default_rng(0)

# Bernoulli data with a Beta prior.  Small n so the difference between CI and credible interval is visible.
alpha0, beta0 = 2, 2
n = 20
data = rng.binomial(1, 0.3, size=n)
k = data.sum()

# Posterior: Beta(alpha0 + k, beta0 + n - k).
alpha_post = alpha0 + k
beta_post  = beta0 + n - k

# 95% credible interval (equal-tail): the 2.5 and 97.5 percentiles of the posterior.
cred_low, cred_high = beta_dist.ppf([0.025, 0.975], alpha_post, beta_post)
posterior_mean = alpha_post / (alpha_post + beta_post)

print(f"Data: k = {k} successes in n = {n} trials.")
print(f"Posterior: Beta({alpha_post}, {beta_post}), mean = {posterior_mean:.4f}")
print(f"95% credible interval: ({cred_low:.4f}, {cred_high:.4f})")
print("Interpretation: given the data and prior, there is a 95% probability that p is in this interval.")

# Frequentist CI using the normal approximation for comparison.
from scipy.stats import norm
p_hat = k / n
se_freq = np.sqrt(p_hat * (1 - p_hat) / n)
z95 = norm.ppf(0.975)
ci_low  = p_hat - z95 * se_freq
ci_high = p_hat + z95 * se_freq
print(f"\\nFrequentist 95% CI (Wald):     ({ci_low:.4f}, {ci_high:.4f})")
print("Interpretation: if we ran this experiment many times, 95% of such CIs would contain the true p.")
print("\\nThe two intervals ARE NOT the same object -- one is Bayesian, one is frequentist.")`;

<Motivation title="Why '87% accuracy' isn't good enough — the missing ± bit">

Say your model got 87% on a benchmark. Great! But here's the honest question: is that really 87%, or could it be as low as 84% or as high as 90%? If you ran the eval again with fresh data, would you get 87% again — or 85%?

That "wobble" around your reported number is called **uncertainty**. Every single measurement in ML has it. Reporting just the point estimate without the wobble is bad science and bad engineering, because:

- 87% accuracy that's really 84–90% is a very different claim from 87% that's really 86.5–87.5%.
- Comparing two models where one got 87% and the other 87.4% is meaningless if both have ±2% wobble.
- Product decisions ("ship model A or model B?") based on tiny differences are just noise unless you know the uncertainty.

The tools for making that wobble explicit are called **intervals**. There are two flavors, and yes, they're annoyingly different despite sounding similar:

- **Confidence interval (CI)** — the frequentist tool. Built from the data such that "if you re-ran the whole experiment many times, 95% of the intervals you'd build this way would contain the true value." The 95% is a promise about your *procedure*, not this specific interval.
- **Credible interval** — the Bayesian tool. An interval where "given the data and your prior beliefs, there's a 95% probability the true value is inside." The 95% is a statement about the *parameter itself*.

Yes, these sound almost the same. They're not, and the distinction matters when writing a paper or defending a product launch. We'll unpack both.

Three payoffs for a GenAI engineer:

- **Every "X% accuracy" number needs a ± after it.** A benchmark result without an error bar is a marketing number, not a scientific claim. Learn to insist.
- **The bootstrap is your Swiss Army knife.** It's a resampling trick that gets you a CI around basically anything — median, correlation, model accuracy, RL return — when the standard formulas either don't exist or don't apply.
- **Credible intervals show up in RLHF, uncertainty estimation, and MC dropout.** When someone claims "the model is 90% confident," ask if that's a Bayesian posterior probability or a frequentist coverage number. They have different meanings and different requirements.

This chapter builds three intervals (formula-based CIs, bootstrap CIs, credible intervals), explains when to use each, and drills the interpretation traps that trip everyone up.

</Motivation>

<Intuition title="Two flavors of interval — one about your procedure, one about the truth">

Let's make the CI-vs-credible distinction click with a real story.

Imagine you're a factory manager checking the true rate of defective widgets. You measure 100 widgets, 4 come out defective, and someone hands you a "95% CI: 1% to 8%".

**Frequentist reading (the CI).** The 95% is a property of *the procedure that made this interval*, not this specific interval. Imagine repeating the whole thing — measuring 100 fresh widgets and recomputing an interval — hundreds of times. About 95% of those intervals would contain the true defect rate. This specific one from today either contains it or doesn't (a fixed fact, unknown to you), but the *long-run track record* of the procedure is 95%.

**Bayesian reading (credible interval).** Now imagine the same interval, but built as a credible interval instead. Then "there's a 95% probability the true defect rate lies between 1% and 8%, given the data and my prior beliefs." That IS a probability about the parameter — the direct answer you actually wanted.

The obvious question: "why can't the CI just say that?" Answer: because in frequentist statistics, the parameter is a fixed unknown number, not a random one. You can't put a probability on a fixed number that's either in the interval or isn't. So the frequentist puts the probability on their *procedure* instead. Bayesians allow the parameter itself to be uncertain (represented by a distribution), which lets them make direct probability statements.

Both are useful. Both are correct within their framework. The two intervals will usually be numerically close for large N with weak priors, so in practice this feels like a philosophy debate more than an engineering one. But if you write a paper or defend a decision, the interpretation matters.

There's also a third thing worth introducing:

**The bootstrap.** It's not a fourth interval — it's a *technique for building intervals*. Here's the idea: resample your data with replacement many times, compute your statistic on each resample, and use the wobble of those results to estimate an interval. It works with almost any statistic (median, correlation, AUC, whatever), and it's often the only realistic way to get error bars on complicated ML measurements.

</Intuition>

<Recall term="Standard error (SE)">

The **standard error** is how much your estimator wobbles across hypothetical fresh samples. Formally, it's the standard deviation of the estimator's sampling distribution. Do not confuse it with the standard deviation of the *data* — SE is about your *estimate*, not your data. For the sample mean X̄ of n iid observations from a population with standard deviation σ: SE(X̄) = σ/√n. That 1/√n shrinkage is why more data → tighter estimates. In practice σ is unknown, so we plug in the sample standard deviation s and call the result the "estimated SE."

</Recall>

<Recall term="Coverage">

**Coverage** is the fraction of hypothetical experiment repetitions in which the CI captures the true value. A well-built 95% CI has coverage around 95%. If yours has coverage of 85%, your interval is too narrow — probably because n is too small, or the data isn't Gaussian, or the samples aren't independent. When people say "the CI is broken," they usually mean it has bad coverage.

</Recall>

## Formula-based CIs — the bell-curve trick

The most-used CI formula in existence comes straight from the **Central Limit Theorem (CLT)**. Vocab first, then formula.

- **CLT** says: no matter what your data looks like (as long as it has finite variance), the sample mean X̄ of a lot of independent samples is approximately bell-curve-shaped around the true mean μ, with a spread of σ/√n. See the CLT chapter for the full story.
- **σ (sigma)** is the true population standard deviation. Unknown in practice.
- **s** is the sample standard deviation, which we compute from our data as a stand-in for σ.

Rearrange the CLT statement, and for large n:

$$
P\left( -1.96 < \frac{\bar X - \mu}{\sigma / \sqrt{n}} < 1.96 \right) \approx 0.95.
$$

Read this as: "the standardized error is within ±1.96 of zero, 95% of the time." Why 1.96? Because for a standard bell curve, that's the range where 95% of the probability lives. If you want a different confidence level, use a different number (e.g., 2.58 for 99%).

Solve for μ inside the inequality and you get a **95% CI for μ**:

$$
\bar X \pm 1.96 \cdot \frac{\sigma}{\sqrt{n}}.
$$

Since σ is unknown, plug in s. But there's a catch: when n is small (say under 30), the "plug in s" trick actually widens the distribution — it becomes something called the **Student's t distribution** (heavier tails than a normal bell curve, but same overall shape). To get correct coverage you use a slightly bigger multiplier — `t_(n-1).ppf(0.975)`, which is about 2.20 for n = 12 and 2.09 for n = 20. For n > 30 the difference basically vanishes and you're back to 1.96.

**Degrees of freedom** = n − 1 in this formula. It's roughly "how many independent pieces of information you have left after you've used one to estimate the mean." Don't panic at the term — it's just an accounting number that comes with the t distribution.

**A few common CI formulas you'll see:**

- **CI for a mean** (unknown σ): X̄ ± t_(n−1) · s/√n. The everyday CI.
- **CI for a proportion** (Wald formula): p̂ ± 1.96 · √(p̂(1 − p̂)/n). *Warning:* this one breaks near p = 0 or p = 1 (can go below 0 or above 1). Better alternatives are called **Wilson score** and **Agresti–Coull** — statsmodels has them under `proportion_confint`.
- **CI for a variance**: uses the chi-squared distribution (χ², "chi-squared" — another distribution shape used for variance stuff). Rarely computed by hand.
- **CI for the difference between two means**: this is the workhorse of A/B testing. Covered in detail in the A/B testing chapter.

## The bootstrap — resample your way to an interval

Here's the idea in one sentence: **treat your observed sample as a mini-universe and draw fresh "samples" from it, with replacement, thousands of times.** The wobble across those pseudo-samples estimates the wobble your real statistic would have across fresh data from the real world. This trick is called the **bootstrap** (Efron, 1979).

The recipe:

1. You have your original sample of size n. Compute your statistic once, call it θ̂ (e.g., the median, the AUC, the accuracy — whatever).
2. Now do this B times (typically B = 1000 or 5000):
   - Draw n items with replacement from the original sample. "With replacement" means the same item can appear multiple times in a resample.
   - Compute the statistic on this resample. Call it θ̂_b.
3. You now have B pseudo-versions of your statistic. Their spread is your estimated sampling distribution.

Three common ways to turn those B pseudo-versions into a 95% CI:

- **Percentile method** — the simplest. CI = (2.5th percentile, 97.5th percentile) of the B values. Great when the pseudo-distribution looks roughly symmetric.
- **Pivot method** — a symmetric-reflection trick that centers on your original θ̂ rather than on the pseudo-distribution's center.
- **BCa (bias-corrected accelerated)** — a smarter percentile method that adjusts for skew and bias. More accurate, more setup. Look this up when you need it; `scipy.stats.bootstrap` implements it.

**Reach for the bootstrap when:**

- Your sample size is small, or your data isn't bell-curve-shaped, so the formula-based CIs give bad coverage.
- Your statistic is complicated — median, correlation, ROC AUC (a common ML metric — see the metrics chapter), quantile — and there's no clean formula for its standard error.
- You don't want to assume a specific distribution shape.

**Bootstrap can mislead when:**

- You're computing an **order statistic** like the max or min. The bootstrap resample max can't exceed the observed max, so the CI's upper end is artificially capped.
- Your data has heavy tails (occasional huge outliers). The bootstrap itself becomes jittery.
- Your sample size is really tiny (n < 20). Resamples are too similar to the original to give useful variation.
- Your data isn't independent (e.g., time series with correlated points). You'd need a variant called the **block bootstrap**.

## Bayesian credible intervals

A **credible interval** is the Bayesian answer. It's an interval [a, b] such that:

$$
P(\theta \in [a, b] \mid \text{data}) = 0.95
$$

Read this as: "given the data (and my prior), the probability that θ sits inside this interval is 95%." That "given the data" is what a frequentist CI can't say — for a frequentist, θ isn't random. For a Bayesian, θ has a distribution called the **posterior**, and the credible interval is just a slice of it.

Two common ways to build a credible interval:

- **Equal-tail credible interval** — chop off 2.5% from each end of the posterior. Easy to compute. This is what most software gives you by default.
- **Highest posterior density (HPD)** — the *shortest* interval containing 95% of the posterior. For symmetric bumps, HPD = equal-tail. For skewed posteriors, HPD is asymmetric and hugs the peak more tightly.

**Fundamental difference vs frequentist CI:** the credible interval is a probability statement *about θ*. You can say "there's a 95% probability the true click-through rate is between 19% and 28%." A frequentist CI cannot say that; it can only say "my procedure produces intervals that contain the truth 95% of the time." Both are useful — just don't confuse the two.

**Nice case: conjugate priors** (see the MAP chapter). If your prior + likelihood is a conjugate pair like Beta + Bernoulli, the posterior is closed-form and you just compute percentiles of a named distribution. Two lines of code.

**Harder case: non-conjugate posteriors.** When the math gets messy, you use approximation techniques. Two names to recognize:

- **MCMC** = **Markov Chain Monte Carlo** — algorithms that draw samples from the posterior even when you can't write it down. Two flavors people mention: **Hamiltonian Monte Carlo (HMC)** and **NUTS** (No-U-Turn Sampler, a fancy HMC variant). Libraries: pyro, pymc, numpyro, stan.
- **Variational inference (VI)** — fit a simple, tractable distribution to approximate the true posterior. Faster than MCMC but less accurate.

Once you have posterior samples (from MCMC) or a fitted approximation (from VI), you compute a credible interval by taking percentiles just like the bootstrap case.

**When the two intervals disagree:**

- Small n with a strong prior. Credible interval is pulled toward the prior; frequentist CI ignores it.
- Highly skewed sampling distribution. Formula-based CI stays symmetric around the estimator; credible interval bends with the skew.
- Different definitions of "the parameter." Bayesian analysis forces you to be explicit about what you're estimating; frequentist analysis can duck this.

In modern practice with moderate sample sizes, the two intervals typically agree to two decimal places. The disagreement matters when it matters (small n, strong prior, tight decision).

## Things that trip people up

**"A 95% CI does NOT mean '95% probability the truth is here.'"** This is the number-one misconception. For a specific CI, the truth is either in it or not — no probability left. The 95% is about the *procedure* over hypothetical repetitions, not about your specific interval. Say it out loud and it sinks in eventually.

**"A credible interval CAN say that."** That's actually the whole point of Bayesian framing. If you want to say "95% probability the true value is in this range," build a credible interval.

**"The bootstrap is not magic."** It's an approximation. It can be biased for order statistics like the max, break with heavy-tailed data, and be worse than a formula-based CI when the formula's assumptions really do hold. Use bootstrap when the formula doesn't fit — not as a universal replacement.

**"Percentile bootstrap gets skew wrong."** When the bootstrap distribution is visibly skewed, use the pivot method or BCa instead of the percentile method.

**"CIs from wrong models are meaningless."** If your data isn't independent, or the distribution assumption is off, the coverage guarantee breaks. Sanity-check with a bootstrap or plot residuals.

**"Wald CIs for proportions break near 0 and 1."** The formula p̂ ± z · √(p̂(1 − p̂)/n) can spit out intervals that go below 0 or above 1 — which is nonsensical for a proportion. Use Wilson or Agresti–Coull when p̂ is close to 0 or 1, or when n is small.

**"Making 20 CIs? Expect one to lie."** Each 95% CI has a 5% chance of missing the truth. If you make 20 of them, you'd expect about one to miss just by chance. If you need joint 95% coverage across all 20, use a wider multiplier (called a **Bonferroni correction**, about 3.0 for 20 intervals instead of 1.96). Same problem as multiple hypothesis testing — covered in the next chapter.

<RealWorldML title="Error bars in ML — benchmarks, RL, uncertainty, A/B, LLM evals">

**1. Benchmark accuracy.** When a paper says "our model got 87.3% on GLUE" (**GLUE** = General Language Understanding Evaluation, a common NLP benchmark), the honest version is "87.3% ± 0.4%" or an explicit CI. Where does that ± come from? Model per-example correctness as a coin flip: each example is either right (1) or wrong (0). Then the CI for accuracy p̂ is √(p̂(1−p̂)/N) times a multiplier. For GLUE with about N = 3,000 examples and accuracy near 87%, SE is around 0.6%. So a "new model got 87.7% vs old 87.3%" claim without any CI could easily be pure noise. This is why serious leaderboards report error bars, and why single-shot benchmark comparisons on small test sets are often misleading.

**2. Reinforcement learning evaluation.** In RL, agents are usually judged by average "return" (reward per episode) across a bunch of runs. But rewards can vary wildly episode-to-episode, especially in games or robotics. Running an agent for just 10 episodes gives you a huge SE. The bootstrap is the fix: take your 10 returns, resample with replacement 5000 times, get a CI on the mean. Every RL paper that says "our method beats prior work" based on 3 seeds and no CI is doing worse science than it looks.

**3. Uncertainty quantification in production.** When your model outputs a probability like "85% chance this email is spam," you often want to know how *sure* the model is about that 85%. Two tools that are essentially bootstrap-in-disguise:

- **MC Dropout** — leave dropout turned on at inference time and run the model 20 times. Different neurons drop out each time, so you get 20 slightly different predictions. The spread of those predictions estimates the model's uncertainty. Cheap, easy to add to any existing model.
- **Deep ensembles** — train 5–10 copies of your model with different random seeds; average their predictions. Their disagreement measures uncertainty. Expensive to train, but very effective — often the most reliable uncertainty estimate you can get.

Both are used in practice to detect **out-of-distribution inputs** (inputs unlike the training data), where models are much more likely to be confidently wrong.

**4. A/B testing.** Every A/B testing platform (Optimizely, Split, in-house tools at Netflix / Meta / etc.) reports a CI or credible interval for the lift ("how much better is B than A"). Two flavors:

- **Frequentist:** two-sample test → CI on the difference of conversion rates.
- **Bayesian:** Beta prior on each arm's conversion rate → posterior samples → credible interval on the difference.

The Bayesian version is often preferred when the team wants to peek at the experiment daily — it handles the "multiple looks" problem better (details in the A/B testing chapter next).

**5. LLM evaluation.** When benchmarking LLMs, you evaluate on N examples and get an accuracy or preference rate. Bootstrap CIs on those numbers give you defensible ranges. This is exactly what the **LMSYS Chatbot Arena** (an LLM head-to-head leaderboard) and many other eval platforms compute behind the scenes. Their **Elo ratings** (a head-to-head ranking system originally from chess) come with bootstrap-derived CIs so you can see which models are statistically distinguishable.

</RealWorldML>

<CodeItYourself title="Three drills — CI coverage, bootstrap for the median, credible-vs-CI comparison">

1. **CI coverage for a proportion.** Simulate 2000 samples of size n = 50 from Bernoulli(0.1). For each, build a Wald 95% CI. Count how many contain 0.1. You should see coverage substantially below 95% — the Wald CI is broken near p = 0. Repeat with the Wilson score CI (`statsmodels.stats.proportion.proportion_confint(..., method='wilson')`) and note the improvement.

2. **Bootstrap CI for the median.** Draw 30 samples from an exponential distribution with rate λ = 1. Compute the sample median. Bootstrap 5000 times, take the 2.5 and 97.5 percentiles of the resample medians. Compare to a normal-approximation CI (there&rsquo;s no clean CLT for the median unless you know the density at the true median — bootstrap wins here).

3. **CI vs credible interval for a small-n Bernoulli.** With n = 15 Bernoulli(0.4) samples, compute (a) a Wald 95% CI, (b) a Wilson score 95% CI, and (c) a 95% Bayesian credible interval with a Beta(1, 1) prior. Which is widest? Which contains the truth if you re-run 100 times? For small n and informative priors, credible intervals can be tighter but rely on the prior being reasonable.

</CodeItYourself>

<Summary>

- **Frequentist 95% CI**: covers the true parameter in 95% of hypothetical repetitions. NOT a probability about a specific interval.
- **Bayesian 95% credible interval**: has 95% posterior probability of containing the true parameter. IS a probability about the parameter.
- **Parametric CI for a mean**: X̄ ± t_(n−1) · s/√n (or z · s/√n for large n).
- **Bootstrap**: resample with replacement B times; percentile / pivot / BCa methods construct CIs from the empirical distribution.
- **Bootstrap strengths**: no distributional assumptions, works for complicated statistics.
- **Bootstrap weaknesses**: order statistics, heavy tails, very small n.
- **Common misinterpretation**: &ldquo;95% CI means 95% chance the truth is in the interval&rdquo; — wrong for CIs, right for credible intervals.
- **Every ML measurement needs an error bar** — accuracy, RL return, A/B lift, benchmark score. Reporting without one is not scientific.

</Summary>
