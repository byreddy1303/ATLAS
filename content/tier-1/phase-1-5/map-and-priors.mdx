export const mapVsMleCode = `# MAP = MLE + a prior term.  For Bernoulli(p) with a Beta(alpha, beta) prior,
# the MAP estimate is (k + alpha - 1) / (n + alpha + beta - 2).
# Compare MLE and MAP when data is sparse.
import numpy as np
from scipy.stats import beta as beta_dist
rng = np.random.default_rng(0)

true_p = 0.05        # rare event (5% success rate)
alpha0, beta0 = 2, 20    # prior belief: mean = 2/22 ~= 9%, moderately confident

print(f"True p = {true_p}, prior = Beta({alpha0}, {beta0}), prior mean = {alpha0/(alpha0+beta0):.4f}")

# Show how MLE vs MAP behave at various sample sizes.
for n in [5, 20, 100, 1000]:
    k = rng.binomial(n, true_p)
    p_mle = k / n
    # MAP for Bernoulli with Beta prior:
    p_map = (k + alpha0 - 1) / (n + alpha0 + beta0 - 2)
    # Posterior mean (Bayesian point estimate, uses full posterior):
    p_post_mean = (k + alpha0) / (n + alpha0 + beta0)
    print(f"n = {n:>4}, k = {k:>3}:  MLE = {p_mle:.4f},  MAP = {p_map:.4f},  posterior mean = {p_post_mean:.4f}")

print("\\nWith small n, MLE is unstable (0 successes in 5 trials -> p_hat = 0);")
print("MAP pulls the estimate toward the prior mean.  With large n, both converge.")
print("This is regularization in the pure statistics sense.")`;

export const l2RidgeCode = `# L2 regularization = Gaussian prior on weights.
# Minimize (1/2) * ||y - X w||^2 + (lambda / 2) * ||w||^2.
# This IS the negative log-posterior under y | X, w ~ N(X w, sigma^2 I) and w ~ N(0, tau^2 I),
# with lambda = sigma^2 / tau^2 -- so shrinking lambda toward zero = wide prior = MLE.
import numpy as np
from sklearn.linear_model import LinearRegression, Ridge
rng = np.random.default_rng(0)

# Ill-conditioned regression problem: 100 samples, 20 features, but only 5 features
# are truly non-zero.  Add moderate noise.
n, d = 100, 20
X = rng.normal(size=(n, d))
true_w = np.zeros(d)
true_w[:5] = np.array([2.0, -1.5, 0.8, -0.6, 1.2])
y = X @ true_w + rng.normal(0, 0.3, size=n)

# Ordinary least squares -- unregularized MLE.
w_mle = LinearRegression(fit_intercept=False).fit(X, y).coef_

# Ridge -- MAP with a Gaussian prior on w.  The lambda coefficient controls prior strength.
w_ridge_small = Ridge(alpha=0.01, fit_intercept=False).fit(X, y).coef_
w_ridge_large = Ridge(alpha=10.0, fit_intercept=False).fit(X, y).coef_

for name, w in [("true w", true_w), ("MLE (OLS)", w_mle), ("Ridge (lambda=0.01)", w_ridge_small), ("Ridge (lambda=10)", w_ridge_large)]:
    print(f"{name:>20}: ||w||_2 = {np.linalg.norm(w):.4f}")

print("\\nHow well does each fit UNSEEN data?")
X_test = rng.normal(size=(2000, d))
y_test = X_test @ true_w + rng.normal(0, 0.3, size=2000)
for name, w in [("MLE (OLS)", w_mle), ("Ridge (lambda=0.01)", w_ridge_small), ("Ridge (lambda=10)", w_ridge_large)]:
    mse = ((X_test @ w - y_test) ** 2).mean()
    print(f"  {name:>20}: test MSE = {mse:.4f}")

print("\\nModerate lambda -> smaller ||w||, better test MSE (bias-variance tradeoff).")
print("The Gaussian prior is doing shrinkage: pulling all coefficients toward zero.")`;

<Motivation title="Every 'regularization' trick is actually a prior in disguise">

**Regularization** is a word thrown around a lot in ML. It just means: "extra terms added to the loss that keep the model from going crazy." Weight decay, L2, L1, dropout, label smoothing — all regularization. Sounds like a bag of unrelated tricks, right?

Here's the neat truth: they're all the same underlying idea. Every common regularizer is a **prior belief** about what "reasonable" parameters look like, written in a way the optimizer can use. Once you see this, the whole toolkit collapses into one concept.

Quick vocabulary before we start, so nothing feels scary:

- **Prior**: your belief about what the parameters *probably* are before you see any data. "Small weights are more sensible than huge weights" is a prior.
- **Likelihood**: how well each parameter setting explains the data you saw. (This came from the previous chapter.)
- **Posterior**: your updated belief after seeing data. Bayes' rule combines prior and likelihood into this.
- **MAP** ("maximum a posteriori"): the single most-probable parameter under the posterior — the peak. Think of it as MLE with a nudge from your prior.

Now the punchline:

- **L2 regularization** (adding λ‖w‖² to the loss) = a Gaussian prior on the weights. In English: "I believe weights should probably be near zero." Every AdamW `weight_decay` parameter in every transformer is this.
- **L1 regularization** (adding λ‖w‖₁) = a Laplace prior. Laplace is a heavy-tailed distribution with a sharp spike at zero. This spike is *why* L1 zeros out coefficients — it prefers "exactly zero" over "small."
- **Dropout, label smoothing, data augmentation, early stopping** — all reinterpretable as priors. Not always exactly, but close enough that the intuition transfers.

Three payoffs for a GenAI engineer that matter right now:

- **Weight decay in transformer training.** Every big model (GPT, Llama, Claude family) trains with a `weight_decay` value in AdamW. That's a Gaussian prior on all billion+ parameters. Without it, transformers memorize instead of generalize.
- **KL penalty in RLHF and DPO.** When fine-tuning a language model with human feedback, you add a "don't drift too far from the original model" term. That's literally a prior on the fine-tuned model, pointed at the pretrained one.
- **Classifier-free guidance** in diffusion image generation. The strength knob γ ("guidance scale") on Stable Diffusion is dialing up how strongly the class label prior influences generation.

MAP is the bridge between two views. On one side: **MLE**, where you just fit the data as well as you can. On the other side: **full Bayesian inference**, where you carry around a whole probability distribution over parameters. MAP is the middle ground — you keep the prior, but summarize the answer with a single "best" number. Get comfortable with MAP and you can walk between both sides.

</Motivation>

<Intuition title="Two voices arguing: what the data says + what you already believed">

Picture this. A coworker slaps a coin on the table and says, "I just flipped this 10 times and got 7 heads. So it's biased, right?"

Two voices immediately pop up in your head:

- **Voice 1 (the data):** "7 out of 10 is more than half. Maybe the coin *is* biased toward heads."
- **Voice 2 (your prior belief):** "But basically every coin I've ever handled is fair. Getting 7-out-of-10 by pure chance isn't even that rare."

Both voices have a point. MAP is just the mathematical way to combine them fairly.

- Voice 1 is the **likelihood** — the score each candidate coin-fairness gets based on how well it explains the observed 7-out-of-10.
- Voice 2 is the **prior** — your belief about coins before you saw this data.

**Bayes' theorem** is the recipe for merging the two into a single updated belief called the **posterior**:

$$
p(\theta \mid D) \;=\; \frac{p(D \mid \theta) \cdot p(\theta)}{p(D)}.
$$

Read out: "the posterior over θ given the data D equals (likelihood × prior) divided by a normalizer." That normalizer p(D) is the same for every θ, so when we're just looking for the peak, we can ignore it.

Take the log (same reason as the last chapter — turns product into sum, easier math):

$$
\log p(\theta \mid D) \;=\; \underbrace{\log p(D \mid \theta)}_{\text{likelihood: data speaks}} + \underbrace{\log p(\theta)}_{\text{prior: you speak}} + \text{const}.
$$

**MAP** is the θ that maximizes this. Think of it as a tug-of-war: the likelihood term pulls θ toward whatever fits the data best; the prior term pulls θ toward whatever you already believed. Where they balance is your answer.

**MLE (last chapter) is just MAP with a flat prior** — meaning "before data, every θ looks equally plausible to me." When the prior is flat, its log is a constant and drops out, leaving only the likelihood. That's why MLE was our starting point.

One important note. MAP gives you a single number: the peak of the posterior. That's fine for training a model. But if you want to say things like "how confident am I in this estimate?", you need the whole posterior distribution, not just its peak. Full Bayesian inference gives you the whole thing; MAP is the fast summary.

</Intuition>

<Recall term="Prior">

A **prior** is your best guess about the parameters *before* looking at the data. It takes the form of a probability distribution p(θ). Two flavors: **uninformative** priors are basically shrug emojis — "any value looks equally likely" (a flat distribution). **Informative** priors carry real belief — "weights are probably near zero," "success rates are probably near 50%." Every Bayesian answer depends on which prior you pick, which is a feature, not a bug — it makes your assumptions explicit.

</Recall>

<Recall term="Posterior">

The **posterior** p(θ | D) is your updated belief about θ *after* seeing data D. It's what Bayes' rule gives you: (likelihood × prior) / normalizer. It's still a distribution — not a single number — because you carry uncertainty. MAP just picks the peak of this distribution for use as a point estimate.

</Recall>

## MAP = MLE + a nudge from the prior

Given data D and a model with parameter θ, MAP picks the θ that maximizes (log-likelihood + log-prior):

$$
\hat\theta_{\text{MAP}} \;=\; \arg\max_\theta \big[ \log p(D \mid \theta) + \log p(\theta) \big].
$$

Depending on what you plug in for the prior, this becomes something you already recognize:

- **Flat prior** (uniform, "no opinion"): the second term is constant → drops out → MAP = MLE.
- **Bell-curve prior** 𝒩(0, σ²) (i.e., "weights should probably be small"): the log-prior is −||θ||²/(2σ²) + const → this is **L2 regularization**.
- **Laplace prior** (spike at zero, heavy tails): the log-prior is −||θ||₁/b + const → this is **L1 regularization**.
- **Peaked prior at some θ₀**: pulls MAP toward θ₀ when the data is thin.

**When the sample size is small.** MAP is much more stable than MLE because the prior gently pulls the estimate away from crazy answers. Example: you flip a coin 5 times and get 0 heads. MLE says "the coin has 0% chance of heads." That's a broken answer — one glance at the coin and you'd disagree. MAP with a Beta(2, 2) prior gives you (0 + 1)/(5 + 2) ≈ 0.14, which is much saner. The prior kept you out of the "degenerate zero" trap.

**When the sample size is large.** The likelihood term grows with n, while the prior term stays fixed. So the data eventually drowns out the prior, and MAP quietly converges to MLE. This is the Bayesian version of "with enough data, your starting beliefs stop mattering."

## L2 regularization = a bell-curve prior on the weights

Let's connect regularization to Bayes on a concrete example: linear regression. Setup:

- You have y (a vector of true values) and X (a matrix where each row is one input).
- Model: y = Xw + noise, where the noise is bell-curve-shaped 𝒩(0, σ²I).
- Goal: find good weights w.

MLE finds the w that minimizes the sum of squared errors (that's just squared distance, ‖y − Xw‖²):

$$
\frac{1}{2\sigma^2} \sum_i (y_i - x_i^\top w)^2 \;=\; \frac{1}{2\sigma^2} \| y - Xw \|^2.
$$

Now add a **prior** on the weights: 𝒩(0, τ²I). In plain English: "before seeing data, I believe each weight is probably small and near zero." That prior's log is:

$$
\log p(w) \;=\; -\frac{1}{2\tau^2} \|w\|^2 + \text{const}.
$$

To find MAP, we minimize the negative-log-posterior (which is likelihood + prior, both negated):

$$
\frac{1}{2\sigma^2} \| y - Xw \|^2 + \frac{1}{2\tau^2} \|w\|^2.
$$

Multiply through by σ² and give the ratio a name — let **λ = σ²/τ²** ("lambda," the regularization strength):

$$
\boxed{\;\text{MAP objective} \;=\; \frac{1}{2} \| y - Xw \|^2 + \frac{\lambda}{2} \|w\|^2\;}
$$

Congratulations — you just derived **ridge regression**. That's a fancy name for "least squares with a squared-weight penalty." The knob λ is a trade-off: λ = 0 means "trust the data" (MLE); λ → ∞ means "trust the prior" (all weights get pushed to zero). Cross-validation is how you pick the sweet spot.

**Where you actually meet this in ML:** every deep-learning optimizer's `weight_decay` parameter. In PyTorch:

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=..., weight_decay=1e-2)
```

That `weight_decay=1e-2` is literally λ. Every LLM (GPT, Llama, Claude family) trains with this. The Gaussian prior is why weights stay reasonable even with billions of parameters.

## L1 regularization = a Laplace prior (and why it zeros things out)

Now swap the bell-curve prior for a **Laplace prior** on each weight. Laplace is another distribution — think of it as "a bell curve that came to a sharp point at zero and has heavier tails." Density:

$$
p(w_j) \;=\; \frac{1}{2b} \exp(-|w_j| / b).
$$

The b is a spread parameter, similar to σ for a Gaussian. Log-prior:

$$
\log p(w) \;=\; -\frac{1}{b} \sum_j |w_j| + \text{const} \;=\; -\frac{1}{b} \|w\|_1 + \text{const}.
$$

That ‖w‖₁ is the **L1 norm** — just "sum of absolute values of the weights." Plug this into the MAP objective, let λ = σ²/b again, and you get:

$$
\text{Lasso objective} \;=\; \frac{1}{2} \|y - Xw\|^2 + \lambda \|w\|_1.
$$

This is **Lasso regression** ("lasso" is the historical name; it stands for "least absolute shrinkage and selection operator" — you don't need to memorize that). What makes it special: L1 zeros out weights that don't matter, giving you a **sparse** solution ("sparse" = "mostly zeros").

**Why does L1 zero things out but L2 doesn't?** Here's the intuition. Both priors want weights near zero. But the Laplace prior has a sharp *spike* at zero — its log has a corner right there, and the derivative is jumpy across it. That corner creates a "dead zone" where the optimizer literally chooses w = 0 exactly. The Gaussian prior is smooth all the way through, so it can shrink weights *toward* zero but almost never *to* zero.

**Where you'll see this:** Lasso is used when you suspect only a few features actually matter (say, thousands of possible medical markers but only 10 that predict a disease). It doubles as a feature selector, since it kills the useless ones. In transformers, L1 shows up less often than L2 (weight decay is the workhorse), but you'll spot it in things like sparse-attention losses and MoE (Mixture-of-Experts) routing penalties, where you want a small subset of things to activate.

## Conjugate priors — the "math stays clean" special case

Sometimes the universe is nice to you. If the prior and the likelihood are picked from the right pair of distributions, then the posterior turns out to be the *same family* as the prior — just with updated numbers. When that happens, the prior is called **conjugate** to the likelihood.

Why does anyone care? Because it means you can update your beliefs with a simple arithmetic step instead of doing a big numerical integration. No fancy sampling, no complicated math — just "add these numbers, divide by these."

The most-used pair: **Beta prior + Bernoulli likelihood**. Say you start with a Beta(α, β) prior for a coin's probability of heads. Two knobs α and β control the shape. Now you flip the coin n times and see k heads. The posterior is:

$$
p \mid D \sim \text{Beta}(\alpha + k, \beta + n - k).
$$

That's it. Add the successes (k) to α, add the failures (n − k) to β. Done. From this you can pull out both the posterior mean and MAP in closed form:

$$
E[p \mid D] = \frac{\alpha + k}{\alpha + \beta + n}, \quad \hat p_{\text{MAP}} = \frac{\alpha + k - 1}{\alpha + \beta + n - 2}.
$$

You can think of α and β as "imaginary prior samples" — Beta(3, 7) is like "before I saw any real data, I had already imagined 3 heads and 7 tails." Watching real data update α and β makes Bayes feel intuitive.

Other conjugate pairs work the same way (their names, so you recognize them, but you don't need to memorize the formulas):

- **Dirichlet + Multinomial**: the multi-class version. Prior for K-class probabilities is Dirichlet; likelihood is Multinomial (which is just "counts across K classes"); posterior is another Dirichlet.
- **Gaussian + Gaussian**: prior on a mean is Gaussian, likelihood is Gaussian → posterior is Gaussian with a weighted-average mean.
- **Gamma + Poisson**: prior on a rate is Gamma; likelihood is Poisson (event-counting distribution); posterior is Gamma.
- **Inverse-Gamma + Gaussian variance**: prior on a variance is Inverse-Gamma → clean posterior.

You won't see conjugate priors much in modern deep learning — models like transformers are far too complicated for closed-form posteriors, so people use approximations like **variational inference** ("fit a simple distribution to the posterior") or **MCMC** ("Markov chain Monte Carlo — draw samples from the posterior"). But conjugate priors are everywhere in classical Bayesian stats and quietly justify the regularizers you meet every day.

## Things that trip people up

**"MAP is not the posterior mean."** They agree when the posterior is symmetric and has one peak. But for skewed posteriors, the peak (MAP) sits in one place and the average (mean) sits somewhere else. If you care about the "expected value" under squared-error losses, use the posterior mean, not MAP.

**"MAP shifts if you change parameterization."** This one bites people. Say your parameter is σ (a standard deviation) and someone reparameterizes to log σ. The MLE gives you the same physical answer either way (that's called *invariance*). MAP does not — the mode of the distribution moves when you transform the variable, because the transformation stretches the density differently in different places. Moral: always be explicit about *which* parameterization your prior is on.

**"Flat priors on infinite ranges aren't really priors."** "Flat over all real numbers" is not a probability distribution because it doesn't integrate to 1. These are called **improper priors**. Sometimes the posterior they induce is still fine, but it's cleaner to use a proper (but wide) prior instead. Just pick a Gaussian with a big variance and move on.

**"You still have to tune the regularization strength."** λ is a **hyperparameter** — a knob that isn't learned from data automatically. Too high → the model underfits (ignores signal). Too low → it overfits (memorizes noise). Cross-validation is how you pick it in practice. Typical values for weight decay in transformers: 1e-2 to 1e-4.

**"Priors are subjective (and that's actually fine)."** Different priors give different answers, especially when data is thin. Bayesians embrace this — the prior is where you get to be honest about what you already know. Frequentists sometimes object. If it worries you, do a **sensitivity analysis**: try a few different priors and see how much the answer changes. If it barely moves, you're fine. If it swings wildly, the data isn't strong enough to make a confident claim yet.

**"MAP is not full Bayes."** MAP gives you one number. Full Bayesian inference gives you a distribution over all plausible θ values. If you need real uncertainty (error bars, calibration, "how sure am I?"), MAP alone isn't enough — you need the whole posterior.

<RealWorldML title="Priors everywhere: how modern LLMs quietly use Bayes">

Let's look at five real training tricks and see the prior hiding inside each one.

**1. Weight decay in transformer pretraining.** Every modern LLM (GPT, Llama, Claude family) trains with a `weight_decay` value like 0.1 in AdamW. That's a **Gaussian prior** on every model parameter — billions of them — saying "weights should probably be small." Why so strong? Because these models are enormous, and without a strong "keep it simple" nudge, they'd just memorize. The prior is what makes them generalize instead of overfit.

**2. KL penalty in RLHF and DPO.** Some vocab first, so this doesn't sound scary:

- **RLHF** = **Reinforcement Learning from Human Feedback**. The process of teaching a language model to prefer answers humans like, using scores/rankings people gave.
- **DPO** = **Direct Preference Optimization**. A simpler alternative to RLHF that skips the reinforcement learning bit.
- **KL divergence** = a number measuring how different two probability distributions are.
- **Policy** = the language model's "way of choosing what to say" — mathematically, the probability distribution it puts on next tokens.
- **π_θ** ("pi-theta") = the current fine-tuned policy. **π_ref** = the reference (pretrained) policy.

When fine-tuning, RLHF and DPO both add a term like β · KL(π_θ ‖ π_ref) to the loss. In English: "penalize any drift from the pretrained model, weighted by β." That is *literally a prior on policies* — you're saying "before fine-tuning, I believed the good policy is close to the pretrained one." β is the prior strength. Too small: the model drifts far, potentially into gibberish or hallucinations. Too large: it barely learns anything new.

**3. Classifier-free guidance (CFG) in diffusion image generators.** In Stable Diffusion (and cousins), there's a knob called **guidance scale** (γ, gamma). Higher γ = the image sticks closer to the text prompt; lower γ = the image is more free-form. Under the hood, γ is dialing up the strength of the "class label prior" during sampling. γ = 1 is the standard conditional generation; γ > 1 is "sharpen the conditioning."

**4. Label smoothing.** When you train a classifier, the usual target for the true class is 1 and everything else is 0. **Label smoothing** softens this — put 0.9 on the true class and spread 0.1 across the others. Result: the model becomes less overconfident and better calibrated. Bayesian view: this is a **Dirichlet prior** on the label distribution — you're saying "even the 'wrong' classes are not zero-probability."

**5. Dropout as approximate Bayesian inference.** **Dropout** = "randomly zero out some neuron outputs during training." Turns out (Yarin Gal, 2016) that dropout is quietly doing approximate Bayesian inference. If you leave dropout on *at inference time* and average predictions across multiple runs, you get useful uncertainty estimates — this is called **MC Dropout** ("Monte Carlo" dropout).

Big theme: **every "regularization" or "auxiliary loss" in modern ML has a Bayesian interpretation waiting behind it.** You don't have to use that interpretation to train the model. But keeping it in your head tells you *why* each trick works and *when* to reach for it.

</RealWorldML>

<Summary>

- **MAP** = argmax_θ (log-likelihood + log-prior). Bayes&rsquo; theorem in log-space, with the evidence dropped.
- **MLE is MAP with a flat prior**. Adding an informative prior changes the estimator; more data eventually washes it out.
- **L2 regularization = Gaussian prior on parameters**. λ (weight-decay strength) = σ²/τ² where σ² is noise variance and τ² is prior variance.
- **L1 regularization = Laplace prior on parameters**. Spike at zero → sparse (exactly-zero) coefficients.
- **Conjugate priors** give closed-form Bayesian updates: Beta–Bernoulli, Dirichlet–Multinomial, Gaussian–Gaussian, Gamma–Poisson.
- **MAP small-N**: robust, stable. Large-N: MAP → MLE.
- **Every ML regularization is a prior**: weight decay = Gaussian, sparsity = Laplace, KL penalty = policy prior, label smoothing = Dirichlet, dropout ≈ variational posterior sampling.

</Summary>
