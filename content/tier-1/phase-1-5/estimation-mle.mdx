# Compare two estimators of the mean of iid samples from a distribution.
import numpy as np
rng = np.random.default_rng(0)

# True population: N(mu = 5, sigma^2 = 4).
true_mu = 5.0
n = 30

print(f"  variance of estimator                                    = {np.var(medians):.4f}")

efficiency = np.var(means) / np.var(medians)

export const mleBernoulliCode = `# MLE for Bernoulli(p): derive by hand, then verify numerically.
# Data: n coin flips, k of which are heads (successes).
# L(p) = product of p^(x_i) * (1-p)^(1-x_i) = p^k * (1-p)^(n-k)
# log L(p) = k * log(p) + (n - k) * log(1 - p)
# d/dp log L = k/p - (n-k)/(1-p) = 0
# =>  k(1-p) = (n-k)p  =>  k - kp = np - kp  =>  k = np  =>  p = k / n.
# MLE for a Bernoulli's success probability is just the empirical frequency.
import numpy as np
from scipy.optimize import minimize_scalar
rng = np.random.default_rng(0)

# Simulate a biased coin with true p = 0.35, observe 100 flips.
true_p = 0.35
n = 100
data = rng.binomial(1, true_p, size=n)
k = data.sum()

# Closed-form MLE.
p_mle = k / n
print(f"Data: {k} heads in {n} flips.")
print(f"Closed-form MLE: p_hat = k/n = {p_mle:.4f}")

# Verify by numerical optimization of the negative log-likelihood.
def neg_log_likelihood(p):
    if p <= 0 or p >= 1:
        return np.inf
    return -(k * np.log(p) + (n - k) * np.log(1 - p))

result = minimize_scalar(neg_log_likelihood, bounds=(1e-6, 1 - 1e-6), method="bounded")
print(f"Numerical MLE (via scipy):                {result.x:.4f}")
print(f"True parameter (would be unknown in practice): {true_p}")

# The MLE is a random variable -- its value depends on the sample.
# For large n, MLE for Bernoulli has variance p*(1-p)/n (Cramer-Rao bound).
CRB_var = true_p * (1 - true_p) / n
print(f"\\nCramer-Rao lower bound for Var(p_hat): {CRB_var:.6f}")

# Verify empirically by repeating the experiment 5000 times.
mles = [rng.binomial(1, true_p, size=n).sum() / n for _ in range(5000)]
print(f"Empirical Var(p_hat) over 5000 experiments: {np.var(mles):.6f}   (matches CRB up to sampling noise)")`;

<Motivation title="Every ML training loop is maximum likelihood estimation">

Maximum likelihood is the single most important idea in Phase 1.5. Almost every loss function you meet in ML is a negative log-likelihood in disguise:

- **MSE regression** = MLE assuming Gaussian noise with fixed variance. Minimizing Σ (yᵢ − ŷᵢ)² *is* maximizing the log-likelihood of the data under y_i ~ 𝒩(ŷ_i, σ²).
- **Cross-entropy classification** = MLE assuming Categorical outputs. Minimizing −Σ log p(y_i) *is* maximizing the log-likelihood of the labels under the model&rsquo;s Categorical distribution over classes.
- **VAE ELBO** = a lower bound on the marginal log-likelihood of the data. Diffusion models are trained with a related bound. Both are MLE-in-spirit.
- **Language-model pretraining** = MLE. Next-token cross-entropy is the negative log-likelihood of each token under the model&rsquo;s Categorical over the vocabulary. GPT&rsquo;s objective is `L = -sum log P(x_t | x_<t)` — a direct chain-rule log-likelihood.
- **Contrastive losses (InfoNCE, SimCLR)** = MLE of a classifier that distinguishes positive from negative pairs. Reformulate and you find likelihood underneath.

Understanding MLE means understanding *why* these losses are the right objectives — not just accepting them as recipes. It also tells you the properties you get for free: consistency (with enough data, the estimator finds the truth), efficiency (near-optimal use of the samples), and asymptotic normality (predictable error bars).

This chapter derives MLE for the two most-used distributions (Bernoulli, Gaussian) by hand, shows why MLE is what every gradient step in ML is climbing, and gives you the vocabulary to spot MLE hiding inside a research paper you read next month.

</Motivation>

<Intuition title="MLE = 'pick the parameter that makes the data you actually saw the least surprising'">

Concrete first. You flip a coin 10 times and see 7 heads. What&rsquo;s a reasonable guess for the true probability of heads?

Try a few candidates. If the true probability were p = 0.5, then P(7 heads in 10 flips) = C(10, 7) · 0.5⁷ · 0.5³ ≈ 0.117. If instead p = 0.7, P(same outcome) = C(10, 7) · 0.7⁷ · 0.3³ ≈ 0.267. That&rsquo;s more than 2× more likely under p = 0.7 than under p = 0.5. Now try p = 0.9: P = C(10, 7) · 0.9⁷ · 0.1³ ≈ 0.057. So among &#123;0.5, 0.7, 0.9&#125;, the observation is most probable under p = 0.7.

The **maximum likelihood estimator** is the p that maximizes this &ldquo;probability of the data.&rdquo; If we search over every p in [0, 1] instead of just three values, we&rsquo;ll find that p̂ = 7/10 = 0.7 exactly. That&rsquo;s the MLE — the value under which the observed data was least surprising.

Generalizing: you&rsquo;ve chosen a distribution family (Gaussian, Bernoulli, Categorical, etc.) — that&rsquo;s the *model*. Its shape depends on a parameter θ. Different values of θ predict different amounts of probability (or density) for the specific data D you observed. **Maximum likelihood picks the θ that assigns highest probability to the observed data.**

Formally, the *likelihood function* L(θ) = P(D | θ) is a function of θ with D fixed. The MLE is

$$
\hat\theta_{\text{MLE}} \;=\; \arg\max_{\theta}\; L(\theta) \;=\; \arg\max_\theta \sum_i \log p(x_i \mid \theta).
$$

We work with the **log-likelihood** because it turns a product over samples into a sum, which is numerically stable (products of tiny probabilities underflow to zero) and easier to differentiate. The log is monotonic, so the argmax is unchanged. Almost everything downstream operates on log-likelihood.

The picture: L(θ) plotted as a function of θ has a peak somewhere; MLE finds that peak. Statistical properties of the estimator (variance, bias, uncertainty) come from how *sharp* that peak is (its second derivative, formally called the *Fisher information*).

</Intuition>

<Recall term="Estimator">

An estimator T is a function of the data (T = T(X_1, …, X_n)) that produces a guess for a population parameter θ. It is itself a random variable — its value depends on which sample you happened to draw. The distribution of T over hypothetical resamples is called its **sampling distribution**. Bias = E[T] − θ. Variance = Var(T). Mean squared error = Bias² + Variance.

</Recall>

## Estimators: bias, variance, consistency, efficiency

An **estimator** is a rule for turning data into a guess about an underlying parameter. Different estimators can target the same parameter — they have different quality characteristics:

- **Bias** = E[T] − θ. An estimator is *unbiased* if its expected value equals the true parameter. Sample mean is unbiased for μ; sample variance with denominator N is biased-low (needs N − 1 to be unbiased).
- **Variance** = Var(T). How much does the estimator swing across resamples? Smaller = more precise.
- **Consistency** = as N → ∞, T converges in probability to θ. All useful estimators are consistent; inconsistent ones are broken.
- **Efficiency** = ratio of your estimator&rsquo;s variance to the minimum-possible variance (the *Cramér–Rao lower bound*). An estimator is *efficient* if it achieves the bound. MLEs are asymptotically efficient — they achieve the Cramér-Rao lower bound in the large-N limit.
- **Robustness** = how much bad data (outliers) breaks the estimator. Median is robust; mean is not.

**Mean squared error decomposition**: MSE(T) = Bias(T)² + Var(T). This is the same identity from the expectation-variance chapter, now applied to estimators. Reducing bias often increases variance and vice versa — a classical bias-variance trade-off.

## The likelihood function

Fix a parametric family (Bernoulli(p), Gaussian(μ, σ²), Categorical(π), …) and iid data D = (x_1, …, x_n). The **likelihood function** is

$$
L(\theta) \;=\; p(D \mid \theta) \;=\; \prod_{i=1}^n p(x_i \mid \theta).
$$

*L is a function of θ, with D fixed.* When p is a PDF (continuous case), L can be greater than 1 — remember, densities aren&rsquo;t probabilities. What matters is *comparing* L across values of θ.

The **log-likelihood** is

$$
\ell(\theta) \;=\; \log L(\theta) \;=\; \sum_{i=1}^n \log p(x_i \mid \theta).
$$

Three reasons we work in log-space:

1. **Numerical stability.** Multiplying n densities produces astronomically small numbers that underflow to zero. Adding n log-densities produces a finite number.
2. **Simple derivatives.** Log of a product = sum of logs. Derivative of a sum = sum of derivatives.
3. **Additivity across iid samples.** ℓ(θ) is a sum of per-sample contributions, matching the mini-batch structure of stochastic gradient descent.

## Deriving MLE for Bernoulli — by hand

Data: n coin flips with outcomes x_i ∈ &#123;0, 1&#125;. Let k = Σ x_i be the number of heads. Model: X_i ~ Bernoulli(p) iid.

$$
L(p) \;=\; \prod_{i=1}^n p^{x_i} (1-p)^{1-x_i} \;=\; p^k (1-p)^{n-k}.
$$

Take log:

$$
\ell(p) \;=\; k \log p + (n - k) \log(1 - p).
$$

Differentiate with respect to p and set to zero:

$$
\frac{d \ell}{d p} \;=\; \frac{k}{p} - \frac{n - k}{1 - p} \;=\; 0
$$

Cross-multiplying: k(1 − p) = (n − k) p, so k = np, and

$$
\boxed{\;\hat p_{\text{MLE}} \;=\; \frac{k}{n}\;}
$$

— just the empirical frequency of heads. Obvious in hindsight; but the MLE derivation is your check that "obvious" is actually correct. **This is also the naive Bayes classifier&rsquo;s per-class word probability, and the standard n-gram counts in language modeling** — same MLE, different context.

## Deriving MLE for Gaussian — the ML regression foundation

Data: n samples x_i ~ 𝒩(μ, σ²) iid. Log-likelihood:

$$
\ell(\mu, \sigma^2) \;=\; -\frac{n}{2} \log(2\pi \sigma^2) - \frac{1}{2\sigma^2} \sum_i (x_i - \mu)^2.
$$

Take partial derivatives.

**MLE for μ**:

$$
\frac{\partial \ell}{\partial \mu} \;=\; \frac{1}{\sigma^2} \sum_i (x_i - \mu) \;=\; 0 \;\Rightarrow\; \boxed{\;\hat \mu_{\text{MLE}} \;=\; \bar x\;}
$$

The sample mean. Independent of σ² — even if you don&rsquo;t know σ², the MLE for μ is still the sample mean.

**MLE for σ²**:

$$
\frac{\partial \ell}{\partial \sigma^2} \;=\; -\frac{n}{2\sigma^2} + \frac{1}{2\sigma^4} \sum_i (x_i - \hat\mu)^2 \;=\; 0
$$

Solving:

$$
\boxed{\;\hat \sigma^2_{\text{MLE}} \;=\; \frac{1}{n} \sum_i (x_i - \hat \mu)^2\;}
$$

**The MLE for variance uses N, not N − 1.** So the MLE for σ² is biased-low; the *unbiased* estimator uses (N − 1) (Bessel&rsquo;s correction). MLE is asymptotically unbiased but small-N biased for variance.

**The ML connection**: linear regression via least squares is *exactly* MLE assuming Gaussian noise. If y_i = f(x_i, w) + ε_i with ε_i ~ 𝒩(0, σ²), then the log-likelihood is

$$
\ell(w) = -\frac{n}{2} \log(2\pi\sigma^2) - \frac{1}{2\sigma^2} \sum_i (y_i - f(x_i, w))^2.
$$

Maximizing over w is equivalent to *minimizing* Σ (y_i − f(x_i, w))² — the squared-error loss. Every MSE loss you write down is quietly assuming Gaussian noise around your predictions.

## MLE properties for large N

MLEs have three beautiful asymptotic properties, all of which are why we use them:

1. **Consistency**: $\hat\theta_{\text{MLE}} \to \theta$ in probability as n → ∞. Given enough data, MLE finds the truth.
2. **Asymptotic normality**: $\sqrt{n} (\hat\theta_{\text{MLE}} - \theta) \to \mathcal{N}(0, I(\theta)^{-1})$, where I(θ) is the **Fisher information**. So for large n, $\hat\theta$ is approximately Gaussian around the true value, with variance approximately 1 / (n · I(θ)).
3. **Asymptotic efficiency**: MLE achieves the Cramér–Rao lower bound in the limit. No other consistent estimator has smaller variance.

The **Fisher information** is the expected curvature of the log-likelihood: $I(\theta) = -E[\partial^2 \ell / \partial \theta^2]$. Intuitively, a sharply-peaked log-likelihood has high curvature → high Fisher information → low estimator variance (we&rsquo;re very certain about θ). Flat log-likelihood → low Fisher information → high variance.

For engineering purposes, this justifies the practice of using MLE and reporting error bars using estimated Fisher information (or its equivalent, the Hessian at the MLE). Every optimizer in `scipy.optimize` and every deep-learning training loop is implicitly finding this peak.

## The traps that matter

**Likelihood is not probability.** L(θ | D) is a function of θ, not a probability distribution over θ. Sums or integrals over θ don&rsquo;t equal 1. You cannot say &ldquo;this θ has probability 0.9 of being right&rdquo; from L alone — that&rsquo;s a Bayesian statement requiring a prior.

**MLE can be sensitive to model misspecification.** If your assumed distribution family doesn&rsquo;t contain the true distribution, MLE finds the θ that&rsquo;s closest (in KL sense) — but that&rsquo;s not the same as the truth. Real data usually violates our model assumptions; check the fit.

**Zero-probability observations kill MLE.** If your model assigns zero density to any observed x_i, log-likelihood → −∞. In practice: use additive smoothing (Laplace, add-α) for count models, ensure your distribution family covers all possible observations, or use a mixture that has global support.

**MLE for variance is biased.** As shown, MLE uses N in the denominator; unbiased uses N − 1. For small N (say &lt; 30) this matters. Deep-learning training often uses MLE variance (or a fixed variance) without concern because N is large.

**MLE can overfit.** For flexible models (deep nets), MLE alone will fit training data too well. Regularization (next chapter: MAP with priors) or early stopping restrains this.

**Numerical implementation matters.** Working in log-space isn&rsquo;t optional. `log_softmax` + `nll_loss` is more stable than `softmax` + `log`. `logsumexp` is more stable than `log(sum(exp(...)))`. Modern libraries have these built-in for a reason.

<RealWorldML title="Every ML training objective is a maximum-likelihood problem in disguise">

Take three canonical setups and unpack the likelihood:

**Cross-entropy classification.** Model outputs π(x, w) = softmax(logits), a Categorical over K classes. Given labeled data (x_i, y_i) with y_i ∈ &#123;1, …, K&#125;, likelihood is:

$$
L(w) = \prod_i \pi_{y_i}(x_i, w).
$$

Log-likelihood:

$$
\ell(w) = \sum_i \log \pi_{y_i}(x_i, w).
$$

Negative log-likelihood is the standard *cross-entropy loss*. Minimizing it *is* MLE for the parameters w of the classifier under the assumption that outputs are Categorical.

**Language model pretraining.** Model outputs a Categorical over the vocabulary V given the preceding tokens. Chain-rule joint over a sentence x = (x_1, …, x_T):

$$
L(w) = \prod_t P(x_t \mid x_{<t}, w).
$$

Log-likelihood: L_LM = Σ_t log P(x_t | x_&lt;t, w). Negating gives the token-level cross-entropy. Every LLM pre-training run is MLE on trillions of tokens.

**Diffusion model score matching.** For a noisy image x_t (produced by forward diffusion), the model learns to predict the noise ε. The training loss is E[|ε − ε_θ(x_t, t)|²] = squared-error, which is MLE for Gaussian noise around the prediction. So diffusion training is *also* MLE — a specific parameterization of the Gaussian likelihood on the noise variable.

Recognizing MLE inside a paper unlocks a lot: you can predict properties of the trained model (asymptotically unbiased, low variance), understand why the loss function looks the way it does, and know what happens as you scale data (LLN, CLT applied to the score).

Every ML training loop is a numerical optimization on a log-likelihood surface. Understanding that is the difference between "here&rsquo;s the loss function they gave me" and "here&rsquo;s the likelihood assumption they made."

</RealWorldML>

<Summary>

- **Estimator** = a function of the data that guesses a population parameter. Its distribution over resamples is its sampling distribution; bias, variance, and MSE describe its quality.
- **Likelihood** L(θ) = P(D | θ) is a function of θ with data fixed. **Log-likelihood** ℓ(θ) = Σ log p(x_i | θ) is what we actually optimize.
- **Maximum likelihood estimator** θ̂_MLE = argmax_θ ℓ(θ). Take derivatives, set to zero, solve.
- **Bernoulli MLE**: p̂ = k/n (empirical frequency).
- **Gaussian MLE**: μ̂ = sample mean; σ̂² = (1/n) Σ (x_i − μ̂)² (uses N, biased-low — the unbiased estimator uses N − 1).
- **Every ML loss is MLE**: MSE ↔ Gaussian, cross-entropy ↔ Categorical, LM pretraining ↔ chain-rule Categorical.
- **MLE properties**: consistent, asymptotically normal (variance 1/(n · Fisher info)), asymptotically efficient.
- **Traps**: likelihood ≠ probability; zero-density observations kill ℓ; MLE overfits without regularization (see next chapter: MAP + priors).

</Summary>
