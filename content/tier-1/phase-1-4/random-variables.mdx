export const pmfCode = `# A random variable is a function from the sample space to a number.
# For a DISCRETE random variable, the probability mass function (PMF)
# gives P(X = x) for each possible value x.
import numpy as np

# X = sum of two fair dice.  Compute the PMF by enumeration.
from collections import Counter
from itertools import product

outcomes = list(product(range(1, 7), repeat=2))
sums = [d1 + d2 for d1, d2 in outcomes]
counts = Counter(sums)
pmf = {s: counts[s] / 36 for s in sorted(counts)}

print("PMF of X = d1 + d2:")
for x, p in pmf.items():
    bar = "#" * int(p * 100)
    print(f"  P(X = {x:>2}) = {p:.4f}   {bar}")

# Verify it is a valid PMF: non-negative and sums to 1.
print(f"\\nSum of PMF values = {sum(pmf.values()):.6f}   (must equal 1)")
print(f"All values non-negative? {all(p >= 0 for p in pmf.values())}")

# Compute a probability of an event using the PMF.
# P(X >= 10) = P(X = 10) + P(X = 11) + P(X = 12)
p_at_least_10 = sum(p for x, p in pmf.items() if x >= 10)
print(f"\\nP(X >= 10) = {p_at_least_10:.4f}   (analytical 6/36 = 0.1667)")`;

# P(X = x) is exactly 0 for any specific x; probability comes from integrating
# the density over an interval: P(a <= X <= b) = integral from a to b of f(x) dx.
import numpy as np
from scipy.stats import norm

# Standard normal N(0, 1): f(x) = (1/sqrt(2*pi)) * exp(-x^2 / 2).
x_grid = np.linspace(-5, 5, 1000)

# Numerical integration -- should equal 1.

print(f"Integral of PDF over R (approx by grid) = {area:.6f}   (must equal 1)")

# Probability of an interval.

# (discrete) or a smooth increasing function (continuous), always in [0, 1].

<Motivation title="Random variables turn events into numbers you can compute with">

A random variable X is just a function from the sample space Ω to numbers. It gives you a scalar (or vector) label for every outcome, which is the shape you actually need to compute expectations, losses, distributions of predictions, and every other numerical quantity in ML. Sample-space language talks about events (subsets); random-variable language talks about *the number you get* — and numbers are what our tools eat.

Every distribution you meet in ML lives inside this framework:

- **Softmax output** = a random variable over the vocabulary, with PMF P(next token = wᵢ) equal to the softmax weight of the i-th logit. Every LLM sampling step draws one realization of this RV.
- **Diffusion noise ε ~ 𝒩(0, I)** = a multivariate continuous random variable. The forward diffusion process is a scheduled sequence of noisy random variables; the reverse process learns to invert it.
- **Model prediction error** = a random variable indexed by data point. Its distribution controls loss, calibration, and confidence intervals.
- **Latent code z in a VAE** = a continuous random variable, usually parameterized as 𝒩(μ, σ²) per data point.

You need two representations. For **discrete** RVs (categorical, count, tokenwise), a **probability mass function** P(X = x) that assigns weight to each possible value. For **continuous** RVs (real-valued, on ℝ or ℝⁿ), a **probability density function** f(x) that says how much probability is packed per unit length — and where P(X = x) = 0 for any single point, but ∫ f(x) dx over an interval gives a real probability. The **cumulative distribution function** F(x) = P(X ≤ x) unifies both.

This chapter builds those three tools (PMF, PDF, CDF) plus the change-of-variables formula that underpins every normalizing-flow and diffusion architecture.

</Motivation>

<Intuition title="A random variable is a labeling of outcomes; a distribution is the histogram of those labels weighted by probability">

Picture the sample space Ω one more time. A random variable X paints a number on every outcome. If you draw an outcome from Ω according to the probability measure P, X gives you a number — that number is itself random. The **distribution of X** is the pattern of numbers you&rsquo;d see if you repeated the experiment many times, weighted by their probabilities.

Discrete: the paint job labels outcomes with values from a countable set. The distribution is a histogram — a PMF. Sum of bars = 1.

Continuous: the paint job labels outcomes with real numbers. The distribution is a smooth curve — a PDF. Area under the curve = 1. No single point has positive probability, but intervals do.

The **CDF** F(x) = P(X ≤ x) works for both: how much total probability lies at or below the value x? In the discrete case it&rsquo;s a step function; in the continuous case it&rsquo;s a smooth increasing function. The PDF is the derivative of the CDF (where the derivative exists); the PMF is the size of each jump in the CDF.

</Intuition>

<Recall term="Sample space">

Ω = the set of all possible outcomes. A random variable is a function X: Ω → ℝ. Every event about X (say &ldquo;X > 5&rdquo;) corresponds to a subset of Ω (all the outcomes whose X-value is greater than 5). Probabilities of RV events come from the underlying probability measure on Ω.

</Recall>

## Discrete random variables and the PMF

A **discrete random variable** takes values in a countable set — finite (a die roll) or countably infinite (a Poisson count). Its **probability mass function** is

$$
p(x) \;=\; P(X = x)
$$

for each value x in the support of X. It is a valid PMF iff two things hold:

- **Non-negativity**: p(x) ≥ 0 for every x.
- **Normalization**: $\sum_x p(x) = 1$.

Every probability question about X reduces to a sum of PMF values. P(X ∈ A) = Σ<sub>x ∈ A</sub> p(x). For X = sum of two dice, the PMF is a triangular histogram peaking at 7.

**Common discrete RVs in ML.** Bernoulli (binary label), Binomial (count of successes), Categorical / Multinoulli (one-hot outcome from K classes — softmax lands here), Poisson (event counts per unit time). We&rsquo;ll catalog them in the next chapter.

## Continuous random variables and the PDF

A **continuous random variable** takes values in an uncountable set, typically ℝ or a subset. For any specific value x, P(X = x) = 0 — there are simply too many points, and each carries no mass on its own. Probability instead lives on *intervals*, expressed via a **probability density function** f:

$$
P(a \leq X \leq b) \;=\; \int_a^b f(x) \, dx
$$

A valid PDF satisfies:

- **Non-negativity**: f(x) ≥ 0 for every x.
- **Normalization**: $\int_{-\infty}^{\infty} f(x) \, dx = 1$.

**Density is not probability.** f(x) has units of probability *per unit x*, so it can be greater than 1 (a very narrow distribution has a very tall density). The number you should always ask for is the *integral over an interval*, not the density at a point.

**Common continuous RVs in ML.** Uniform, Gaussian / normal, exponential, log-normal, Beta, Gamma, Dirichlet. All of these are cataloged in the next chapter. The Gaussian in particular is the workhorse: latent codes, weight initializations, gradient noise, most CLT-based analyses.

## The CDF — the universal representation

The **cumulative distribution function** is defined for any RV, discrete or continuous:

$$
F(x) \;=\; P(X \leq x).
$$

Properties:

- Monotone non-decreasing.
- F(−∞) = 0, F(+∞) = 1.
- Right-continuous. For discrete RVs, F is a step function; for continuous, F is (usually) smooth and increasing.
- **PDF is the derivative of the CDF** where it exists: f(x) = F′(x).
- **PMF is the size of each jump** in the CDF at discrete atoms: p(x) = F(x) − F(x⁻).
- **P(a &lt; X ≤ b) = F(b) − F(a)** — the most useful identity for interval probabilities.

The CDF is also the tool for **sampling from an arbitrary distribution**. If U ~ Uniform(0, 1), then X = F⁻¹(U) has CDF F. This inverse-CDF (or &ldquo;quantile&rdquo;) method is how libraries generate samples from any distribution whose CDF you can invert.

## Change of variables — how transformations reshape distributions

If Y = g(X) for a smooth, monotone, invertible function g, the density of Y is

$$
f_Y(y) \;=\; f_X(g^{-1}(y)) \cdot \left| \frac{d g^{-1}(y)}{dy} \right|.
$$

The `|dg⁻¹/dy|` factor is called the **Jacobian correction**. Without it, densities don&rsquo;t transform properly — they wouldn&rsquo;t integrate to 1 after the transform. The intuition: `g` compresses or stretches space; the density has to compensate so that mass is preserved.

**Multidimensional generalization.** For Y = g(X) with X, Y ∈ ℝⁿ:

$$
f_Y(\mathbf{y}) \;=\; f_X(g^{-1}(\mathbf{y})) \cdot |\det J_{g^{-1}}(\mathbf{y})|
$$

where J is the Jacobian matrix of g⁻¹. This determinant term is *exactly* what makes normalizing flows (RealNVP, Glow, IAF) trainable: the flow architecture ensures the Jacobian is easy to compute — typically triangular so its determinant is the product of diagonal entries — while allowing arbitrary invertible transforms of a base distribution into a complex learned one.

**Worked example**: X ~ 𝒩(0, 1), Y = exp(X). Then g⁻¹(y) = log(y) with derivative 1/y. So $f_Y(y) = \frac{1}{y\sqrt{2\pi}} \exp\left(-\frac{(\log y)^2}{2}\right)$ — the log-normal density.

## The traps that matter

**Density is not probability.** A density value f(x) has units of &ldquo;probability per unit x.&rdquo; It can be larger than 1. Never quote f(x) as if it were a probability. Always integrate.

**Careful with discrete vs continuous when mixing.** If your RV has both point-mass atoms (e.g., a zero-inflated model with P(X = 0) &gt; 0) and continuous portion, use the CDF or a mixture representation. Pure PDF/PMF language breaks.

**The Jacobian is mandatory in change of variables.** Every normalizing flow paper starts with `log p(x) = log p_z(z) + log |det J|`. Forget the determinant and your log-likelihoods will be off by an amount that depends on the transform.

**Sampling ≠ evaluating the density.** Some models (VAEs, normalizing flows) let you evaluate the density; some (score-based diffusion) only let you sample. Both are useful; know which one your model supports.

**Support matters.** The support of an RV is the set where its density (or mass) is positive. Log-normal has support (0, ∞); trying to evaluate its density at a negative number gives zero or NaN in a bad library, not a helpful error.

**Continuous CDFs are always continuous; PDFs may not be.** F is smooth iff the underlying density is (roughly) continuous. Distributions with sharp cutoffs (uniform, exponential at zero) have PDFs with jumps but CDFs that stay continuous.

<RealWorldML title="Random variables everywhere: LLM sampling, diffusion noise, VAE latents">

Three canonical GenAI systems, each with a distinct random-variable structure:

**LLM next-token sampling.** At each generation step, the model produces logits over the vocabulary, softmax turns them into a categorical PMF, and the sampled token is a *discrete* random variable with that PMF. Temperature scales the logits (T &lt; 1 sharpens; T &gt; 1 flattens); top-k restricts the support; top-p (nucleus) restricts to the smallest set whose cumulative CDF exceeds p. Every decoding decision is a manipulation of the PMF or its CDF.

**Diffusion models.** The forward process at each step adds Gaussian noise ε ~ 𝒩(0, I): a *continuous* random variable, isotropic, mean zero, with a known PDF. The full noisy image x_t is a random variable whose distribution is a scaled + shifted version of x₀ plus noise. Training is: given a noisy x_t, predict the noise ε (equivalently, the score ∇ log p(x_t)). At sampling time, you invert this process — Markov chain of RVs going from pure noise back to a clean image.

**VAE latent codes.** For each input x, the encoder outputs μ(x) and log σ²(x); the latent z is sampled as z = μ(x) + σ(x) · ε with ε ~ 𝒩(0, I) — the *reparameterization trick* that makes gradients flow through the sampling operation. z is a *continuous* random variable whose density is a Gaussian centered at μ(x). The decoder maps z back to reconstruction. The whole framework rests on interpreting z as an RV and its distribution as a learnable posterior q_ϕ(z | x).

The unifying principle: **every generative model is a change of variables from a simple base RV (Gaussian, uniform, or categorical) to a complex target distribution.** Diffusion learns a stochastic change of variables; flows learn a deterministic invertible one; VAEs learn a conditional latent variable. Change-of-variables + Jacobian corrections + density evaluation are the mathematical toolkit that runs the whole show.

</RealWorldML>

<Summary>

- **Random variable** X: Ω → ℝ. Turns outcomes into numbers you can compute with.
- **Discrete RVs** have a **PMF** p(x) = P(X = x). Non-negative, sums to 1. Every probability is a sum of PMF values.

- **Every generative model** is a change of variables from a base RV to a complex target: flows (deterministic invertible), VAEs (learned Gaussian latents), diffusion (stochastic Markov chain).

</Summary>
