export const entropyBasicCode = `# Entropy in bits: measures the "spread" of a probability distribution.
# High entropy = uncertain, spread out; low entropy = concentrated, predictable.
import numpy as np

def entropy_bits(p):
    p = np.asarray(p, dtype=float)
    p = p[p > 0]                        # avoid 0 * log(0) (defined as 0)
    return float(-np.sum(p * np.log2(p)))

# fair 8-sided die: max entropy at log2(8) = 3 bits
p_fair8   = np.ones(8) / 8
# very skewed: almost always outcome 0
p_skewed  = [0.9, 0.02, 0.02, 0.02, 0.02, 0.02, 0.0, 0.0]
# perfectly certain: 100% outcome 0
p_certain = [1.0, 0, 0, 0, 0, 0, 0, 0]

for name, p in [("uniform (8 outcomes)", p_fair8),
                ("skewed",               p_skewed),
                ("certain (P=1 on one)", p_certain)]:
    print(f"H({name:>22}) = {entropy_bits(p):.3f} bits")`;

export const crossEntropyCode = `# Cross-entropy: the loss LLMs (and any classifier) minimize.
# H(p, q) = - sum p * log q   — average bits needed to encode samples from p using code for q.
# When p is a one-hot target (say token index 3 is correct), H reduces to -log(q[3]).
import numpy as np

def cross_entropy(p, q, eps=1e-12):
    p, q = np.asarray(p, float), np.asarray(q, float) + eps
    return float(-(p * np.log(q)).sum())

# Vocabulary of 8 tokens; the true next token is index 3 (one-hot target)
target_onehot = np.zeros(8); target_onehot[3] = 1.0

# Model predictions (softmax outputs — probability over vocab)
q_confident_right = [0.02, 0.02, 0.02, 0.86, 0.02, 0.02, 0.02, 0.02]  # peaks at 3
q_uniform         = np.ones(8) / 8                                     # no info
q_confident_wrong = [0.02, 0.02, 0.02, 0.02, 0.02, 0.86, 0.02, 0.02]  # peaks at 5 (WRONG)

for name, q in [("confident right", q_confident_right),
                ("uniform",         q_uniform),
                ("confident WRONG", q_confident_wrong)]:
    ce = cross_entropy(target_onehot, q)
    print(f"CE({name:>16}) = {ce:.3f} nats  = -log(q[3]) = {-np.log(q[3] + 1e-12):.3f}")`;

export const klDivergenceCode = `# KL divergence: how many EXTRA bits/nats you pay if you use q's code for p's distribution.
# KL(p || q) = sum p * log(p/q).  Always >= 0.  Zero iff p == q.  NOT symmetric.
import numpy as np

def kl_div(p, q, eps=1e-12):
    p, q = np.asarray(p, float), np.asarray(q, float) + eps
    return float((p * (np.log(p + eps) - np.log(q))).sum())

# Two similar distributions
p = np.array([0.5, 0.3, 0.2])
q = np.array([0.4, 0.4, 0.2])
print(f"KL(p||q) = {kl_div(p, q):.4f} nats")
print(f"KL(q||p) = {kl_div(q, p):.4f} nats    <- NOT the same as KL(p||q)!")
print()

# Zero KL only when distributions are identical
r = np.array([0.5, 0.3, 0.2])
print(f"KL(p||r) = {kl_div(p, r):.4f} nats    <- identical, KL is 0")
print()

# CE = H(p) + KL(p||q)  — cross entropy decomposes into entropy plus divergence
def entropy_nats(p, eps=1e-12):
    p = np.asarray(p, float); p = p[p > eps]
    return float(-(p * np.log(p)).sum())

def cross_entropy_nats(p, q, eps=1e-12):
    p, q = np.asarray(p, float), np.asarray(q, float) + eps
    return float(-(p * np.log(q)).sum())

print("Verifying the identity  CE(p, q) = H(p) + KL(p||q):")
print(f"  H(p)            = {entropy_nats(p):.4f}")
print(f"  KL(p||q)        = {kl_div(p, q):.4f}")
print(f"  sum             = {entropy_nats(p) + kl_div(p, q):.4f}")
print(f"  CE(p, q) direct = {cross_entropy_nats(p, q):.4f}")`;

<Motivation title="The LLM loss is cross-entropy — this chapter says WHY">

Every large language model on earth minimizes **cross-entropy** during training. It's not a coincidence, and it's not just "the standard classification loss." Cross-entropy is the *unique* loss that turns a probability model into an optimizer with the right calibration properties, and its close cousin the **KL divergence** underlies half of modern deep learning: RLHF for LLMs, distillation for compression, variational autoencoders for generation, contrastive learning for embeddings.

For a GenAI engineer, information theory is the single highest-leverage math chapter — one theorem in this chapter (cross-entropy = entropy + KL divergence) explains:

- **Why we train with -log(p_correct_token) per token.** That's cross-entropy against a one-hot target — exactly what PyTorch's `nn.CrossEntropyLoss` computes.
- **Why perplexity is the standard eval metric.** Perplexity = exp(cross-entropy per token) = "effective vocabulary size the model is uncertain over." Lower perplexity = sharper model.
- **How RLHF's KL penalty prevents mode collapse.** RLHF adds a KL(π || π_ref) term to keep the fine-tuned model close to the reference; if the KL grows, the model has drifted from natural language.
- **Why knowledge distillation works.** Student trained on teacher's softmax output = minimize KL(teacher || student), aka "match the teacher's probability distribution" instead of just top-1.
- **What temperature really is in sampling.** Softmax with temperature T = distribution with entropy scaled by T; low T = confident, high T = high-entropy (creative).

Master this chapter and you can *read* every loss function in every modern paper without looking anything up. That's the payoff for a couple hours of study.

</Motivation>

<Intuition title="Information = bits of surprise. Entropy = average surprise. Cross-entropy = surprise using the wrong code.">

Rare events are informative; common events are not. If someone tells you "the sun rose this morning," the information content is ~0 bits — you already knew it. If they tell you "the sun didn't rise this morning," that's an enormous amount of information — you'd never predicted it.

**Shannon information** of an outcome with probability p is defined as I(p) = −log(p). Small p (rare event) → large I (lots of information). Large p (common event) → small I (little information). If we use log base 2, I is in **bits**; if we use natural log (base e), it's in **nats** — a scaling constant, same idea.

**Entropy** is the *expected* information content of a random draw from a distribution: H(p) = E[I(X)] = -Σ p_i log p_i. High-entropy distributions are unpredictable (uniform); low-entropy distributions are concentrated (nearly deterministic).

**Cross-entropy** H(p, q) = -Σ p_i log q_i measures the average surprise if you *believe* q is the distribution but samples actually come from p. It's minimized when q = p (no surplus surprise), and it always ≥ H(p). The gap is called **KL divergence** — the extra bits/nats you pay for using the wrong distribution.

That's the entire chapter. Everything below is turning these three mental pictures into formulas, code, and neural-network practice.

</Intuition>

## Shannon information and entropy

The information content of an outcome with probability p is:

$$
I(p) \;=\; -\log_2 p \quad \text{(bits)} \qquad \text{or} \qquad -\ln p \quad \text{(nats)}
$$

The logarithm base sets the unit (bits, nats, or trits); pick one and stick with it. In ML the natural log is standard because it plays nice with calculus (d/dx ln(x) = 1/x, no log-2 constants everywhere).

**Entropy** of a discrete distribution p = (p_1, …, p_K) over K outcomes:

$$
H(p) \;=\; -\sum_{i=1}^{K} p_i \log p_i
$$

with the convention 0 · log(0) = 0 (limit as p → 0⁺). Properties you should burn in:

- **Non-negative**: H(p) ≥ 0. Zero iff one outcome has probability 1 (perfect certainty).
- **Maximum for uniform**: over K outcomes, H is maximized at H = log K when p_i = 1/K for all i. The uniform distribution is the "most uncertain" one — you'd need log K bits on average to communicate the outcome.
- **Concave in p**: mixing two distributions never decreases entropy (mixing adds uncertainty). This is the source of many inequalities and of the *maximum-entropy principle* used in some models.

## Cross-entropy — the LLM loss

Cross-entropy between a "true" distribution p and a "predicted" distribution q, both over the same K outcomes:

$$
H(p, q) \;=\; -\sum_{i=1}^{K} p_i \log q_i
$$

Interpretation: **average information (in bits/nats) needed to encode samples from p using a code optimized for q**. If q = p, you're using the optimal code, and H(p, q) = H(p). If q is wrong, you use a suboptimal code and pay extra — that extra is exactly the KL divergence, defined below.

**The one-hot LLM case.** Suppose the model has to predict the next token, and the true next token is index 3 (out of a vocab of size K). The "true distribution" p is one-hot: p_3 = 1, everything else is 0. The model outputs a softmax distribution q over the vocab. Cross-entropy becomes:

H(p, q) = -Σ_i p_i log q_i = **-log q_3**

— the negative log probability the model assigned to the correct token. That's it. Every "next-token cross-entropy loss" line in every LLM training loop is computing -log(model_probability_of_correct_token), averaged over tokens.

<RunnableCode title="Cross-entropy for one-hot targets — what LLMs minimize" code={crossEntropyCode} packages={["numpy"]} />

Notice how cross-entropy penalizes confident wrong answers *hugely*: a model that assigns 0.02 to the correct token (as in the "confident wrong" case) has CE = -log(0.02) ≈ 3.9 nats, while assigning uniform 1/8 = 0.125 has CE = 2.08 nats. Being confidently wrong is *worse* than being uniformly ignorant — this asymmetry is why LLMs eventually learn to output high probabilities only when they're actually right.

## KL divergence — how far apart are two distributions

The **Kullback–Leibler divergence** from p to q:

$$
D_{\mathrm{KL}}(p \parallel q) \;=\; \sum_{i} p_i \log \frac{p_i}{q_i}
$$

Read: "the average log-ratio of probabilities under p." Equivalent forms you'll see:

- D_KL(p || q) = Σ p log p − Σ p log q = **H(p, q) − H(p)** — cross-entropy minus entropy.
- D_KL(p || q) = E_(p)[log(p/q)] — an expectation under p of the log-ratio.

**Fundamental properties**:

- **Non-negative**: D_KL(p || q) ≥ 0. Zero iff p = q almost everywhere (Gibbs' inequality). Proof: use Jensen's inequality on the convex function -log.
- **Non-symmetric**: D_KL(p || q) ≠ D_KL(q || p) in general. This is *not* a distance metric (violates symmetry and triangle inequality). Some contexts prefer the symmetric variant JSD(p, q) = (KL(p||m) + KL(q||m))/2 with m = (p+q)/2.
- **Not upper-bounded**: KL can be +∞ if q assigns zero probability where p assigns non-zero (any impossible-under-q event that p thinks is possible → infinite surprise). Real code adds a small ε to avoid log(0).

<RunnableCode title="KL divergence — asymmetric, non-negative, zero only at identity" code={klDivergenceCode} packages={["numpy"]} />

**The identity that ties it all together:**

$$
H(p, q) \;=\; H(p) + D_{\mathrm{KL}}(p \parallel q)
$$

— cross-entropy decomposes into "irreducible entropy of the true distribution" + "extra surprise from using the wrong model." Since H(p) doesn't depend on q, **minimizing cross-entropy over q is exactly the same as minimizing KL divergence D_KL(p || q)**. This is the mathematical reason we use cross-entropy loss instead of a more complicated "divergence" loss: minimizing them is equivalent, and cross-entropy is easier to compute (no p log p term needed).

## Perplexity — the LLM eval metric explained

**Perplexity** of a language model on a test corpus:

$$
\mathrm{PPL} \;=\; \exp\left(\frac{1}{N}\sum_{t=1}^{N} -\ln q(x_t \mid x_{<t}) \right) \;=\; \exp(\text{average NLL per token})
$$

Interpretation: **effective vocabulary size** the model is uncertain over at each token. A uniform model over a 50,000-word vocabulary has PPL = 50,000. GPT-3 on WebText has PPL ≈ 20 — meaning per token, the model is roughly as uncertain as if it were choosing uniformly from 20 words. That's a 2500× improvement over uniform, over three years of research from 2018 to 2020, largely from architecture (transformers) and scale (parameters, data, compute).

Perplexity is a direct function of cross-entropy per token, so lowering LM loss lowers perplexity. Everyone reports perplexity because it's more interpretable — "20-word effective vocab" is more intuitive than "3.0 nats per token."

## The traps that matter

**Cross-entropy vs "negative log likelihood" are the same thing** in the discrete-label case. "NLL loss" and "cross-entropy loss" appear side-by-side in ML papers because they're identical when the target is one-hot. Two names, one operation. PyTorch's `nn.NLLLoss` expects log-probabilities as input; `nn.CrossEntropyLoss` expects raw logits and applies log-softmax internally. Same computation.

**Softmax followed by cross-entropy is numerically fused for stability.** Computing softmax then taking log can lose precision (softmax may produce numbers so small that log returns -inf). All ML libraries provide a fused `log_softmax` or `CrossEntropyLoss(logits, target)` op that avoids the intermediate. Use them; don't implement it yourself unless you understand the log-sum-exp trick.

**KL asymmetry matters for variational inference and generative models.** D_KL(p || q) is "forward" KL — mode-covering, tends to make q spread across p's modes. D_KL(q || p) is "reverse" KL — mode-seeking, tends to make q concentrate on one of p's modes. Variational autoencoders and RLHF policies (typically) use reverse KL; density estimation uses forward. Pick the wrong direction and your generative model collapses.

**Temperature-scaled softmax = entropy control**. `softmax(z / T)`: T = 1 is standard, T → 0 makes softmax approach argmax (near-zero entropy, always outputs the max), T → ∞ makes it uniform (max entropy). "Temperature" in LLM sampling directly controls the entropy of the sampling distribution — high T = more diverse/creative outputs (higher perplexity if measured), low T = more deterministic/repetitive.

**RLHF's KL penalty prevents catastrophic forgetting.** The objective is E_π[reward(x)] − β · KL(π || π_ref), where π is the RLHF-tuned policy and π_ref is the pretrained base. Without the KL term, the policy would drift arbitrarily far to maximize reward — including learning to game the reward model or generate incoherent text. The KL term is a "leash" that keeps the fine-tuned model in the neighborhood of natural language. β is a critical hyperparameter — too small = policy collapses to reward hacking; too large = model doesn't learn anything new.

**Distillation uses "soft targets" via KL.** Instead of training a student on hard labels (one-hot), train it on the teacher's softmax output (a proper distribution). The loss becomes KL(teacher || student), often with temperature-scaled softmax to emphasize the ratios between non-top outputs. This "dark knowledge" from the teacher's uncertainty is what makes distilled models retain more accuracy than training a small model from scratch on hard labels.

<RealWorldML title="Cross-entropy IS the language model loss — one loop, one line">

Here's the language model training loss in one PyTorch line: `loss = F.cross_entropy(logits.view(-1, vocab_size), targets.view(-1))`. Under the hood:

1. **`logits`** has shape (batch, seq_len, vocab_size) — raw scores for each vocab word at each position.
2. **`F.cross_entropy`** internally applies log-softmax to convert logits → log-probabilities, then indexes into the log-probability of the *correct* target token at each position (a one-hot cross-entropy), then averages over positions.
3. The gradient of this loss w.r.t. the logits at position t is **q_t − p_t** where q_t is the model's softmax output and p_t is the one-hot target — a beautifully clean gradient signal that says "increase probability of the correct token, decrease probability of all others." No calculus tricks needed; this falls out of the cross-entropy derivative.

Perplexity on validation = `exp(avg_loss_per_token)` — that's your eval number. Sub-20 on WebText = GPT-3 level. Sub-10 on domain-specific corpora is possible; sub-5 on constrained tasks (code, math). These numbers directly measure how well the model's probability distribution matches the true distribution of natural language sequences, per the KL-cross-entropy identity.

For RLHF post-training, the loss becomes the PPO objective with a KL penalty: `loss = -reward * log_prob + beta * KL(policy || ref)`. That KL is exactly the divergence we defined here, computed either exactly (for on-policy samples where both distributions are available) or estimated (via importance sampling). The whole "align to human preferences without forgetting language" story is encoded in choosing the right beta.

Every knob you'll tune while training or fine-tuning an LLM — learning rate, temperature, KL coefficient, distillation weight — connects back to some concept in this chapter.

</RealWorldML>

<CodeItYourself title="Three drills — entropy, cross-entropy in a mini-classifier, RLHF-style KL penalty">

1. **Entropy of a hand-crafted distribution.** Compute H(p) in bits for p = (0.5, 0.25, 0.125, 0.125). Verify by hand: each term is p · log₂(1/p), which equals p · length of Huffman code. Total = 0.5·1 + 0.25·2 + 0.125·3 + 0.125·3 = 1.75 bits. Interpretation: on average, 1.75 bits per symbol are needed — that's what an optimal code achieves (and what H measures).

2. **Cross-entropy in a two-class classifier.** Given true label y = 1 and model prediction q = 0.8 (prob of class 1), compute the binary cross-entropy BCE(y, q) = -[y log q + (1-y) log(1-q)] = -log(0.8) ≈ 0.223. Now try q = 0.5 (BCE = -log(0.5) = 0.693) and q = 0.99 (BCE = -log(0.99) ≈ 0.010). Plot BCE as a function of q for y = 1 — you'll see a curve that grows toward +∞ as q → 0 (confidently wrong is punished hard) and asymptotes to 0 as q → 1.

3. **RLHF-style KL leash.** Given two categorical distributions — a "reference" p_ref = softmax([1, 0, 0]) and a "policy" p_pi = softmax([z, 0, 0]) for varying scalar z — compute D_KL(p_pi || p_ref) as z varies from 0 to 5. Plot the curve. Observe how a small z (close to reference) has small KL, and large z (policy strongly prefers option 0) has large KL. The RLHF training objective would balance this KL against a reward signal.

</CodeItYourself>

<Summary>

- **Information** I(p) = -log p — bits of surprise for an outcome of probability p.
- **Entropy** H(p) = -Σ p log p — expected surprise; measures uncertainty of a distribution. Zero at certainty, max at uniform (log K over K outcomes).
- **Cross-entropy** H(p, q) = -Σ p log q — expected surprise if you use q's code for p's samples. Reduces to -log q[correct] for one-hot targets — the LLM training loss.
- **KL divergence** D_KL(p || q) = Σ p log(p/q) — the "extra bits/nats" you pay for using q instead of the true p. Always ≥ 0, zero iff p = q, NOT symmetric.
- **Key identity**: H(p, q) = H(p) + D_KL(p || q). Minimizing cross-entropy = minimizing KL divergence (since H(p) doesn't depend on q).
- **Perplexity** = exp(average NLL per token) — the effective vocabulary size a language model is uncertain over. Standard LM eval metric.
- **In practice**: PyTorch's `F.cross_entropy` fuses softmax + log + negative-log for stability. Every LLM training loop is one line of this. RLHF adds a KL term against a reference policy to keep the tuned model from drifting.
- **Forward vs reverse KL**: forward (D_KL(p||q)) is mode-covering; reverse (D_KL(q||p)) is mode-seeking. Choice depends on whether you want your model to be inclusive or selective.

</Summary>
