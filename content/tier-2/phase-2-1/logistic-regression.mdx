export const trainLogRegCode = `# Train logistic regression from scratch via gradient descent on the log-loss.
# Same math as sklearn.LogisticRegression -- reduced to bare numpy.
import numpy as np, matplotlib.pyplot as plt

np.random.seed(0)

# Two-cluster 2D classification -- linearly separable with a small overlap
N_per = 150
X_pos = np.random.randn(N_per, 2) + np.array([+1.8, +1.5])
X_neg = np.random.randn(N_per, 2) + np.array([-1.5, -1.2])
X = np.vstack([X_pos, X_neg])
X = np.hstack([X, np.ones((2 * N_per, 1))])                # bias column
y = np.concatenate([np.ones(N_per), np.zeros(N_per)])

def sigmoid(z): return 1 / (1 + np.exp(-z))

def log_loss(w, X, y, eps=1e-12):
    "Binary cross-entropy on the whole dataset."
    p = sigmoid(X @ w)
    return -np.mean(y * np.log(p + eps) + (1 - y) * np.log(1 - p + eps))

def grad_log_loss(w, X, y):
    "d/dw of log-loss is X.T @ (p - y) / N -- the SAME shape as linear regression's gradient."
    p = sigmoid(X @ w)
    return X.T @ (p - y) / len(y)

# Gradient descent
w = np.zeros(3)
lr = 0.5
losses = []
for step in range(500):
    losses.append(log_loss(w, X, y))
    w = w - lr * grad_log_loss(w, X, y)

print(f"Final loss: {log_loss(w, X, y):.4f}")
print(f"Fitted weights: {w.round(3)}")
preds = (sigmoid(X @ w) > 0.5).astype(int)
print(f"Training accuracy: {(preds == y).mean():.4f}")

# Plot 1: decision boundary
fig, axes = plt.subplots(1, 2, figsize=(11, 4.4))
xx, yy = np.meshgrid(np.linspace(-5, 5, 200), np.linspace(-5, 5, 200))
grid = np.hstack([xx.reshape(-1, 1), yy.reshape(-1, 1), np.ones((xx.size, 1))])
probs = sigmoid(grid @ w).reshape(xx.shape)
axes[0].contourf(xx, yy, probs, levels=20, cmap="RdBu", alpha=0.6)
axes[0].contour(xx, yy, probs, levels=[0.5], colors="k", linewidths=2)
axes[0].plot(X_pos[:, 0], X_pos[:, 1], "o", color="crimson", markersize=4, alpha=0.6)
axes[0].plot(X_neg[:, 0], X_neg[:, 1], "s", color="royalblue", markersize=4, alpha=0.6)
axes[0].set_title("Logistic regression decision boundary")
axes[0].set_xlabel("x_1"); axes[0].set_ylabel("x_2"); axes[0].grid(alpha=0.3)

# Plot 2: loss curve
axes[1].semilogy(losses, color="crimson", lw=2)
axes[1].set_xlabel("GD step"); axes[1].set_ylabel("log-loss (log scale)")
axes[1].set_title("Training loss decreases monotonically (convex loss)")
axes[1].grid(alpha=0.3)

export const softmaxCode = `# Multiclass logistic regression = softmax regression.
# Each class gets its own weight vector w_k; probabilities are given by softmax.
import numpy as np, matplotlib.pyplot as plt

def softmax(Z):
    "Row-wise softmax with numerically-stable form (subtract max for stability)."
    Z_shift = Z - Z.max(axis=1, keepdims=True)
    exp = np.exp(Z_shift)
    return exp / exp.sum(axis=1, keepdims=True)

# 3-class synthetic 2D data
np.random.seed(0)
N_per = 100
centers = [np.array([2, 2]), np.array([-2, 2]), np.array([0, -2])]
X_list, y_list = [], []
for c, center in enumerate(centers):
    X_list.append(np.random.randn(N_per, 2) + center)
    y_list.append(np.full(N_per, c))
X = np.vstack(X_list)
X = np.hstack([X, np.ones((3 * N_per, 1))])
y = np.concatenate(y_list)
K = 3    # 3 classes

# One-hot the labels for the loss computation
Y_onehot = np.eye(K)[y]

def train_softmax(X, Y, lr=0.5, steps=200):
    W = np.zeros((X.shape[1], K))
    for _ in range(steps):
        Z = X @ W
        P = softmax(Z)
        grad = X.T @ (P - Y) / len(X)
        W = W - lr * grad
    return W

W = train_softmax(X, Y_onehot)
Z = X @ W
P = softmax(Z)
preds = P.argmax(axis=1)
print(f"Training accuracy: {(preds == y).mean():.4f}")
print(f"Class-3 weight vectors (each column):\\n{W.round(3)}")

# Plot decision regions
fig, ax = plt.subplots(figsize=(6.5, 5.5))
xx, yy = np.meshgrid(np.linspace(-5, 5, 200), np.linspace(-5, 5, 200))
grid = np.hstack([xx.reshape(-1, 1), yy.reshape(-1, 1), np.ones((xx.size, 1))])
grid_preds = softmax(grid @ W).argmax(axis=1).reshape(xx.shape)
ax.contourf(xx, yy, grid_preds, levels=[-0.5, 0.5, 1.5, 2.5], cmap="RdBu", alpha=0.4)
colors = ["crimson", "royalblue", "seagreen"]

<Motivation title="Logistic regression — the bridge from regression to classification (and to every neural net)">

Take linear regression from last chapter, but change what we're predicting. Instead of a continuous number like "house price," predict a **class label** — a discrete category. Common examples:

- Spam vs. not-spam (email filter)
- Positive vs. negative review (sentiment)
- Click vs. no-click (ad targeting)
- Toxic vs. safe (content moderation)

Sounds easy — why not just fit linear regression to 0/1 labels? Turns out that doesn't work well. The predictions escape the [0, 1] range (you might predict a "probability" of 1.7 or −0.3), the loss surface has weird cliffs when you try to threshold, and the whole thing feels bolted together.

**Logistic regression** fixes this cleanly with two moves:

1. Compute the same linear score `z = w · x + b` as before.
2. Squish `z` through a special function called the **sigmoid** (a smooth S-curve — more soon) that maps *any* real number into the interval (0, 1). Now the output is a real probability.

The training objective changes accordingly: instead of MSE, we minimize the **log-loss** (a.k.a. **binary cross-entropy** — both names refer to the same thing). It's convex, has a clean gradient, and — this is the pretty part — it's *exactly* the same objective you'd get from **maximum likelihood** if you assumed each label was a coin-flip with probability `σ(w · x)`.

For a GenAI engineer, logistic regression is *insanely* important:

1. **Every neural classifier ends with it.** The final layer of ResNet, of BERT's classification head, of every transformer producing next-token probabilities — is a linear layer followed by softmax (the multiclass sibling of sigmoid). That's literally softmax regression on top of learned features. If you understand this chapter, you understand what happens at the top of every classification model.
2. **The log-loss shows up everywhere.** Cross-entropy loss for image classification, next-token prediction in LLMs, reward modeling in RLHF — all versions of logistic regression's log-loss. Understand it once here, understand it forever.
3. **Softmax + cross-entropy is one of the most important pairings in deep learning.** Understanding *why* they cancel to give the beautiful gradient `p − y` (predicted minus true) is the key to how classification networks train.
4. **Linear probing works here too.** Fit a logistic regression on top of frozen pretrained embeddings, see how well it separates the classes. That's how a lot of vision-language transfer research is evaluated.

</Motivation>

<Intuition title="Score, squish, threshold — a three-step recipe">

Here's the entire logistic-regression pipeline in three moves:

**Step 1 — Score.** Compute `z = w · x + b`, exactly like linear regression. Think of `z` as an "evidence-for-class-1 meter": large positive `z` means "I'm confident it's class 1," large negative means "I'm confident it's class 0," `z` near zero means "I'm on the fence."

**Step 2 — Squish.** Push `z` through the **sigmoid**, an S-shaped curve that squishes any real number into a probability in (0, 1):

$$
p = \sigma(z) = \frac{1}{1 + e^{-z}}
$$

Behavior worth memorizing: `σ(z) → 0` as `z → −∞`, `σ(z) → 1` as `z → +∞`, and `σ(0) = 0.5`. So `p` is a valid probability, and it hits exactly ½ when the score is zero.

**Step 3 — Threshold (for a hard prediction).** If `p ≥ 0.5`, predict class 1; otherwise class 0. That's equivalent to "predict 1 if `z > 0`."

That's the prediction machinery. Now: how do we *train* it?

**The training goal.** Pick `w` that makes each true label as *likely as possible* under our model. Concretely: for each example, we ask "how much probability did we assign to the actual answer?" We want that to be high for every example.

- If the true class is 1 and we predicted `p`, our probability-of-being-right is `p`. Its `log` is our "score" (using log because probabilities multiply across independent examples, but logs add — much nicer to work with).
- If the true class is 0, our probability-of-being-right is `1 − p`.

Sum those log-probabilities across the dataset, flip the sign so we can *minimize* instead of maximize (a training convention), and divide by `N` for an average. Out pops the **log-loss** (equivalently, **binary cross-entropy**):

$$
\mathcal{L}(w) = -\frac{1}{N} \sum_i \left[ y_i \log p_i + (1 - y_i) \log (1 - p_i) \right]
$$

Read this: for each example, only one of the two terms is "on" (the other is multiplied by zero). If the true label `yᵢ` is 1, we penalize small `pᵢ`; if it's 0, we penalize large `pᵢ`. That's exactly what we want.

Good news: this loss is **convex** (one big bowl, no local traps). So gradient descent from any starting point converges to the unique best answer. Same three-step recipe generalizes to multi-class prediction using **softmax** (later in the chapter).

</Intuition>

<Recall term="Sigmoid function">

The sigmoid is `σ(z) = 1 / (1 + exp(-z))`. Properties: (1) maps `R → (0, 1)`, so its output can be interpreted as a probability. (2) monotonically increasing. (3) `σ(0) = 0.5`. (4) derivative `σ'(z) = σ(z)(1 - σ(z))`, which is easy to compute given σ(z) itself — this is why sigmoids were popular in early neural nets. (5) saturates at both ends (`σ'(±10) ≈ 0`), which is why deep networks with sigmoid activations had vanishing-gradient problems — ReLU replaced sigmoid inside deep nets, but sigmoid is still the standard output for binary classification.

</Recall>

## The model, spelled out

Same building blocks as linear regression, plus one squish:

**The linear score** — nothing new. Given features `x` (of length `D`), weights `w` (also length `D`), and bias `b`:

$$
z = w \cdot x + b
$$

**Pass through the sigmoid to get a probability:**

$$
\hat{p} = \sigma(z) = \frac{1}{1 + e^{-z}}
$$

**Make a hard prediction (if you need one):** if `p̂ ≥ 0.5` predict class 1, otherwise class 0. That's the same as asking "is `z ≥ 0`?" — which means the boundary line between the two classes is exactly:

$$
w \cdot x + b = 0
$$

That's a **hyperplane** (line in 2D, plane in 3D, flat "surface" in higher dimensions). The perceptron uses the same hyperplane. So does an SVM. What makes logistic regression *different* isn't the boundary itself — it's that logistic regression gives you a *probability of being on the class-1 side*, not just a hard yes/no. That extra probability structure is what lets us do MLE, calibrate outputs, and combine predictions with other probabilistic models.

**Bias trick — same as before.** Append a column of 1s to `X` so we can pretend `b` is just another weight. Every formula below assumes we've done this.

## The log-loss — where it comes from, why it's beautiful

We've been calling it "log-loss" like it's a magic incantation, but it isn't. It falls out of one of the cleanest MLE derivations in ML. Let's walk through it.

**Set up the story.** We have `N` examples `(xᵢ, yᵢ)` with each `yᵢ` either 0 or 1. We *model* each label as a coin flip whose bias depends on the input:

$$
y_i \sim \text{Bernoulli}(p_i), \quad p_i = \sigma(w \cdot x_i)
$$

("Bernoulli" is just the fancy name for a single coin flip — see the [random variables chapter](/learn/mathematics/probability/random-variables).)

**Write the likelihood.** The probability that our model would produce the actual labels we saw, given weights `w`, is:

$$
L(w) = \prod_i p_i^{y_i} (1 - p_i)^{1 - y_i}
$$

The trick with the exponents: when `yᵢ = 1`, only the `pᵢ` term survives; when `yᵢ = 0`, only `1 − pᵢ` survives. So each factor equals whichever probability was assigned to the correct answer.

**Maximum likelihood says: pick `w` to make this as big as possible.** But products of many small probabilities go to zero fast. Take the log (which converts products to sums), flip the sign (so we can minimize a *loss* instead of maximizing a likelihood), and divide by `N` for an average:

$$
\mathcal{L}(w) = -\frac{1}{N} \sum_i \left[ y_i \log p_i + (1 - y_i) \log (1 - p_i) \right]
$$

That's the **log-loss**. It has two names — "log-loss" (from the ML tradition) and "binary cross-entropy" (from information theory, where cross-entropy measures how many bits it takes to encode true labels using the model's predicted distribution). Same object.

**Why it behaves so nicely.** Two properties make training reliable:

- **Convex.** The log-loss is convex in `w` — one big bowl, no local minima to get stuck in. Gradient descent from any starting point lands at the same global best `w*`. Sketch of why: compute the Hessian (matrix of second derivatives). It works out to `Xᵀ · D · X`, where `D` is a diagonal matrix with positive entries `pᵢ(1 − pᵢ)`. Any matrix of this "sandwich" shape with positive middle is **positive semi-definite**, which is the linear-algebra way of saying "the loss is convex."
- **Clean gradient.**

$$
\nabla_w \mathcal{L}(w) = \frac{1}{N} X^\top (\sigma(Xw) - y) = \frac{1}{N} X^\top (\hat{p} - y)
$$

Notice: this has *exactly the same shape* as linear regression's gradient `Xᵀ(Xw − y)/N`. The only difference: instead of `Xw` (raw prediction), we have `σ(Xw)` (squished prediction). And the residual `p̂ − y` is bounded in magnitude by 1, which keeps things numerically stable.

This "predicted minus actual" pattern isn't a coincidence — it's a deep property that happens whenever you pair the "right" output activation with its matching loss. Cross-entropy loves sigmoid; MSE loves the identity. Break the pairing and you lose the beautiful gradient.

## Softmax regression — the multi-class version

Real-world problems usually have more than two classes. Image classification has 1,000 (ImageNet). Next-token prediction has 50,000+ (a full vocabulary). We need a way to scale up.

**Enter softmax.** The **softmax function** is sigmoid's multi-class sibling. Give each class `k` its own weight vector `wₖ`, compute a score per class:

$$
z_k = w_k \cdot x
$$

Now the vector `z = (z₁, z₂, …, zₖ)` holds one score per class. Softmax turns those scores into a probability distribution:

$$
p_k = \mathrm{softmax}(z)_k = \frac{e^{z_k}}{\sum_j e^{z_j}}
$$

Two things to notice: (1) every `pₖ > 0` (because `eˣ` is always positive), and (2) they sum to 1 (because the numerator is one term of the sum in the denominator). So it's a valid probability distribution over the classes. Bigger score → bigger probability. Same idea as sigmoid, just with more than two options.

**The loss — categorical cross-entropy.** Same MLE story, generalized:

$$
\mathcal{L}(W) = -\frac{1}{N} \sum_i \log p_{y_i}(x_i)
$$

Read this: for each example, take the probability the model assigned to the *actual* correct class, take its log, sum across examples, negate, and average. If you assigned high probability to the right class, `log p` is close to 0; if you assigned low probability, `log p` is a big negative number — and the loss (which is `−log p`) is a big positive number. The loss punishes confident wrong answers *hard*.

If you write the label as a **one-hot vector** `y` (all zeros except a 1 at the true class), the loss can be written more compactly as `−Σₖ yₖ log pₖ` — same object, different notation.

**The gradient is unreasonably clean.** With softmax outputs `p` and one-hot labels `y`, the gradient of the loss with respect to the *logits* `z` is:

$$
\frac{\partial \mathcal{L}}{\partial z_k} = p_k - y_k
$$

That's it. **Predicted minus true.** Same `p − y` pattern as the binary case. This is why softmax + categorical cross-entropy is the standard classifier output combo — the gradient computation for the last layer of every classification network is one subtraction per class. It's the reason the last few lines of every classifier's backward pass are so short.

**Numerical stability — the max-subtraction trick.** Never compute softmax the naive way — the `eᶻ` term will overflow to infinity if any `zₖ > ~700` in float64. Standard fix: subtract the max score from every entry before exponentiating.

```python
def softmax(z):
    z = z - z.max(axis=-1, keepdims=True)
    e = np.exp(z)
    return e / e.sum(axis=-1, keepdims=True)
```

Why does this work? The subtraction cancels between numerator and denominator: `e^(zₖ − m) / Σ e^(zⱼ − m) = e^zₖ · e^(−m) / (e^(−m) · Σ e^zⱼ) = e^zₖ / Σ e^zⱼ`. Mathematically identical, but every `e^x` argument is now non-positive, so nothing overflows. Every well-implemented softmax uses this trick.

## What the decision boundary looks like

The **decision boundary** is the surface in feature space where the model is equally torn between two classes — the "line in the sand." Understanding its shape tells you what kinds of patterns logistic regression can and can't capture.

**Binary case.** The boundary is the hyperplane `w · x + b = 0`. On one side of it, `z > 0`, so `σ(z) > 0.5`, so we predict class 1. On the other side, class 0. The distance from any point `x` to the boundary is `|w · x + b| / ‖w‖` — bigger distance means more confident prediction.

**Multi-class case.** The boundary between any two classes `j` and `k` is the hyperplane `(wⱼ − wₖ) · x + (bⱼ − bₖ) = 0` — the set of points where those two classes get *equal* score. With `K` classes there are `K · (K−1) / 2` such pairwise boundaries. In practice, each point in feature space just gets assigned to whichever class has the biggest score.

**Three things to keep in mind:**

- **Logistic regression can only draw straight boundaries** in the space you give it. Real-world class boundaries are often curved. To draw a curved boundary, either (a) add nonlinear features by hand (e.g., include `x²`, `x · y`, etc. — the **polynomial features** trick), (b) use the **kernel trick** (implicit high-dim mapping — the SVM tradition), or (c) switch to a nonlinear model (tree, neural net).
- **The boundary depends on all training points, not just the ones near the margin.** SVMs only care about the "support vectors" (points at the margin); logistic regression takes every point into account. That's why you'll see different-looking boundaries for the two.
- **Add features → change the boundary shape.** Adding `x₁²` and `x₂²` lets LR draw ellipses. Adding `x₁ · x₂` lets it draw hyperbolas. Same feature-engineering idea as polynomial regression — you can get any polynomial shape by adding enough features, but the fit is still "linear in the feature space" you constructed.

Here's a claim that ties this whole chapter to modern deep learning: **the final layer of every neural-net classifier is a linear layer followed by softmax.** No exceptions worth mentioning. That last layer *is* softmax regression, sitting on top of whatever "features" the network has learned. Concretely:

- **Vision (ResNet-style).** The convolutional stack encodes an image into a 2048-dimensional **embedding** (a summary vector). Then a linear layer `W ∈ ℝ^(2048 × 1000)` followed by softmax gives probabilities over 1,000 ImageNet classes. That final head is softmax regression.
- **Language (transformer).** The transformer encodes a token into a `d_model`-dimensional vector (typically 768 to several thousand). Then a linear layer `W ∈ ℝ^(d_model × V)` (where `V` is vocabulary size — 50,000+) followed by softmax gives you the probability distribution over the next token. Same head, different classes.
- **Multi-modal (CLIP-style).** Encode image → image embedding. Encode text → text embedding. The "classifier" is cosine similarity between them, which is *literally* a linear-in-embeddings score, followed by softmax over candidate texts. Same story, just with two encoders.

**Why does this matter for a GenAI engineer?** Four practical reasons:

1. **Linear probing is exactly what we just did.** When you evaluate a pretrained model by fitting `sklearn.LogisticRegression` on top of its frozen embeddings — that's replacing the softmax head with a new softmax head, trained only on the task you care about. If the probe fits well, the encoder already captures the target signal.
2. **Fine-tuning = linear probing + also updating the encoder.** During fine-tuning, gradients flow from the classification loss back through the softmax head *and* into the encoder's parameters. Every SFT (supervised fine-tuning) run for an LLM, every ImageNet fine-tune, every RLHF reward-model training run — they're all "logistic regression, plus also update everything below."
3. **Calibration matters and starts from here.** Big classifiers are usually **overconfident** — they'll report 90% confidence on things that actually happen 60% of the time. Fixing that (via **temperature scaling**, **Platt scaling**, or **isotonic regression**) means tweaking the logistic layer's output. Understanding logistic regression is the foundation for calibration work.
4. **The `p − y` gradient is the "backward pass" of every classifier.** Every time PyTorch does `loss.backward()` on a classification model, the very first step it computes is `p − y` — the same predicted-minus-true residual we derived in this chapter. Understand this scalar identity and you understand the *first* step of the backward pass in every classifier you'll ever train.

</RealWorldML>

<Summary>

- **Model:** `p̂ = σ(w · x + b)` where σ is the sigmoid. Threshold at 0.5 to get a binary prediction.
- **Sigmoid:** `σ(z) = 1/(1+e^{-z})`. Smooth, monotonic, maps R to (0, 1). Derivative: `σ'(z) = σ(z)(1 - σ(z))`.
- **Loss:** binary log-loss (= binary cross-entropy = NLL under Bernoulli): `-Σ [y log p + (1-y) log (1-p)] / N`. Convex → GD converges to global optimum.
- **Gradient of log-loss w.r.t. w:** `X^T (σ(Xw) - y) / N` — same form as linear regression's gradient with `Xw` replaced by `σ(Xw)`.
- **Multiclass:** softmax + categorical cross-entropy. Same clean gradient `∂L/∂z_k = p_k - y_k`.
- **Decision boundary is a hyperplane** (linear in feature space). Polynomial features → curved boundaries.
- **Every classifier's last layer is logistic (softmax) regression on top of learned features.** Fine-tuning = softmax head plus encoder update.
- **Numerical stability:** always subtract max before exp in softmax; clip probabilities away from 0 and 1 before log.

</Summary>
