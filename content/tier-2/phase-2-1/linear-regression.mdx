export const closedFormCode = `# Linear regression via the normal equations: fit y = X @ w + noise on synthetic data.
# Closed-form: w_hat = (XᵀX)⁻¹ X^T y  (assuming X^T X is invertible).
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(0)
N, D = 100, 3

# Synthetic ground truth: 3 features + a bias, y = X @ w_true + noise
X_raw = np.random.randn(N, D)
X     = np.hstack([X_raw, np.ones((N, 1))])          # append bias column
w_true = np.array([2.0, -3.0, 0.5, 1.0])             # last is bias
y = X @ w_true + 0.5 * np.random.randn(N)            # sigma = 0.5

# The normal equations, spelled out
XtX = X.T @ X
Xty = X.T @ y
w_hat = np.linalg.solve(XtX, Xty)                    # solve is more stable than @ inv

print(f"true    w = {w_true}")
print(f"fitted  w = {w_hat.round(3)}")
print()

# Sanity check against numpy's canonical solver
w_lstsq, *_ = np.linalg.lstsq(X, y, rcond=None)
print(f"lstsq   w = {w_lstsq.round(3)}   (should match fitted w)")

# Residual sum of squares and R^2
y_hat = X @ w_hat
rss   = ((y - y_hat) ** 2).sum()
tss   = ((y - y.mean()) ** 2).sum()
r2    = 1 - rss / tss
print(f"\\nRSS = {rss:.3f}   TSS = {tss:.3f}   R^2 = {r2:.4f}")

# Plot the fit versus target (2D projection: first feature vs y)
fig, ax = plt.subplots(figsize=(7, 4))
order = np.argsort(X[:, 0])
ax.plot(X[order, 0], y[order], "o", color="royalblue", alpha=0.6, label="data")
ax.plot(X[order, 0], y_hat[order], "-", color="crimson", lw=2, label="fitted line (marginal)")
ax.set_xlabel("feature 0"); ax.set_ylabel("y"); ax.grid(alpha=0.3); ax.legend()
ax.set_title("Linear regression: first-feature projection of the fit")
plt.tight_layout()`;

export const gdVsClosedFormCode = `# Gradient descent finds the SAME solution as the normal equations,
# but iteratively — the two agree in the limit.
import numpy as np, matplotlib.pyplot as plt

np.random.seed(0)
N, D = 300, 4
X = np.hstack([np.random.randn(N, D), np.ones((N, 1))])
w_true = np.array([1.5, -2.0, 0.3, 0.8, 1.0])
y = X @ w_true + 0.4 * np.random.randn(N)

def loss(w): return 0.5 * np.mean((X @ w - y) ** 2)
def grad(w): return X.T @ (X @ w - y) / N

# Closed form
w_star = np.linalg.solve(X.T @ X, X.T @ y)
print(f"Closed-form w = {w_star.round(3)}")
print(f"Closed-form loss = {loss(w_star):.6f}")

# Gradient descent from zero init
w = np.zeros(X.shape[1])
lr = 0.03
losses = [loss(w)]
for step in range(1, 5001):
    w = w - lr * grad(w)
    if step % 500 == 0:
        print(f"step {step:>4}: loss = {loss(w):.6f}  |w - w*| = {np.linalg.norm(w - w_star):.6f}")
    losses.append(loss(w))

print(f"\\nFinal GD  w = {w.round(3)}")
print(f"Distance to closed-form: {np.linalg.norm(w - w_star):.6f}")

plt.figure(figsize=(7, 3.6))
plt.semilogy(losses, color="crimson", lw=2)
plt.axhline(loss(w_star), linestyle="--", color="teal", label=f"closed-form loss ({loss(w_star):.4f})")
plt.xlabel("GD step"); plt.ylabel("loss (log)"); plt.title("GD converges to the closed-form loss")
plt.legend(); plt.grid(alpha=0.3); plt.tight_layout()`;

<Motivation title="Linear regression — the one model to derive fully">

Linear regression is the simplest possible supervised model. Given features `x` and a target `y`, predict:

$$
\hat{y} = w \cdot x + b
$$

That is it. One dot product per prediction. Yet **every single idea in modern ML has a "linear regression version"** — regularization, gradient descent, MLE, feature engineering, cross-validation, bias-variance decomposition, model interpretability, uncertainty estimation. Understanding all of them fully in the linear-regression setting is the cheapest way to build intuition for their harder analogs.

For a GenAI engineer, linear regression comes up in four specific places:

1. **Debugging.** When a big model is misbehaving, one of the fastest sanity checks is "does a linear regression on the same features do reasonable things?" If YES, the signal is there and the big model is broken. If NO, the features themselves don't capture what you need.
2. **Linear probing.** Modern LLM/vision research evaluates pretrained representations by fitting a linear model on top of frozen embeddings. If the linear probe's accuracy is high, the pretrained model already "understands" the target without task-specific fine-tuning.
3. **The last layer of everything.** Almost every neural net ends with a linear layer (`W · h + b` where h is the last hidden state). That's a linear regression (or classification) whose weights are learned jointly with the encoder.
4. **Baseline you must beat.** Before deploying a fancy model, always compare it to linear regression. It's free, fast, and often surprisingly close in accuracy on tabular data — a paper reporting a 5% improvement over a linear baseline on some task is trivial; a paper reporting a 50% improvement is real.

This chapter covers:

- The **model** and its squared-error loss (why squared, and why the classical alternatives lose).
- The **closed-form solution** (the normal equations) — derived by hand.
- The **gradient descent path** to the same answer, and when to prefer it.
- The **four classical assumptions** and how their violations manifest in residual plots.
- **Interpretability of coefficients** — the real reason linear regression is still central to statistics.

</Motivation>

<Intuition title="Fit the best line through the points — but 'best' means smallest total squared error">

Given a scatterplot of `(x_i, y_i)` points, the goal is to draw a line (or a hyperplane in higher dimensions) that "best" passes through them.

**Best = smallest squared error.** For each data point i, the model predicts `ŷ_i = w · x_i + b`. The error is `ŷ_i − y_i`. Squaring the errors (before summing) turns positive and negative errors into "how far off?" and penalizes big errors much more than small ones:

$$
L(w, b) = \frac{1}{N} \sum_{i=1}^N (\hat{y}_i - y_i)^2 = \frac{1}{N} \sum_i (w \cdot x_i + b - y_i)^2
$$

The line that minimizes this is unique (under mild conditions). And crucially, it has a **closed-form solution** — you can write down the optimal `w` and `b` in matrix form, no iterative training needed.

The story extends to multiple features. If x has D dimensions, we're fitting a hyperplane in (D + 1)-space (D features + 1 for the bias). The picture is the same: minimize total squared error, get a unique optimum.

</Intuition>

<Recall term="Mean squared error (MSE)">

Given predictions `ŷ_i` and true values `y_i` for i = 1..N, the **mean squared error** is `(1/N) · Σ (ŷ_i − y_i)²`. It's the loss function used by linear regression and the "standard" loss for continuous-valued targets. Two properties: (1) it's differentiable everywhere (unlike absolute error), so gradient methods work; (2) it corresponds to maximum likelihood under Gaussian noise — a fact the assumptions section formalizes.

</Recall>

<Recall term="Dot product">

For two vectors `a` and `b` of the same length, the dot product is `a · b = Σ_i a_i b_i`. It's the single operation behind linear regression's prediction (`ŷ = w · x`), attention (`softmax(Q·Kᵀ/√d)·V` uses dot products), classification (each output logit is a dot product), and every "score" in ML. When you optimize a linear regression, you're optimizing a set of dot-product-produced predictions.

</Recall>

## The model and its loss

**Model.** For a single example with features `x ∈ R^D`:

$$
\hat{y} = w_1 x_1 + w_2 x_2 + \dots + w_D x_D + b = w \cdot x + b
$$

For a batch of N examples with feature matrix `X ∈ R^{N × D}` and targets `y ∈ R^N`:

$$
\hat{y} = X w + b \mathbf{1}
$$

**The bias trick.** Absorb `b` into `w` by appending a column of 1s to X: `X_aug = [X | 1]`, `w_aug = [w; b]`. Then `ŷ = X_aug · w_aug` and the bias drops out of every subsequent formula. In code:

```python
X_aug = np.hstack([X, np.ones((N, 1))])
```

**Squared-error loss (halved for clean derivative):**

$$
L(w) = \frac{1}{2N} \|Xw - y\|^2 = \frac{1}{2N} \sum_{i=1}^N (\hat{y}_i - y_i)^2
$$

Halving isn't necessary — it just makes the gradient prettier. Any positive scalar multiple gives the same optimizer.

**Why squared and not absolute error?** Three reasons:

1. **Differentiable everywhere.** Absolute error `|ŷ - y|` has a kink at zero; MSE is smooth. Gradient methods work directly on MSE.
2. **Closed-form solvable.** MSE gives linear equations for the optimum; MAE requires linear programming or iteratively-reweighted-least-squares.
3. **Statistically justified.** Under Gaussian noise assumption `y = w·x + ε` with `ε ~ N(0, σ²)`, the MLE for w minimizes exactly the MSE. Different noise models give different losses (Laplace noise → MAE, etc.).

## The normal equations — closed-form solution

Set the gradient of the loss to zero:

$$
\nabla_w L(w) = \frac{1}{N} X^\top (X w - y) = 0
$$

Solve for w:

$$
X^\top X w = X^\top y \quad \Longrightarrow \quad w^* = (X^\top X)^{-1} X^\top y
$$

This is **the normal equations**. Two remarks:

- **Uniqueness requires rank(X) = D** (X has full column rank). Then `X^T X` is invertible. If features are perfectly correlated (multicollinearity), `X^T X` is singular and the solution is not unique.
- **Numerical stability.** Never actually compute `inv(X^T X)`. Instead use `np.linalg.solve(X.T @ X, X.T @ y)` or better `np.linalg.lstsq(X, y)` — both are far more numerically stable than explicit inversion.

<RunnableCode title="Fit linear regression via normal equations" code={closedFormCode} packages={["numpy", "matplotlib"]} captureFig={true} />

**Cost of closed form:** computing `X^T X` is O(N·D²). Solving the D×D system is O(D³). For small D (say D under 10,000) this is instant. For D ≥ 10⁶, it becomes prohibitive — that's when GD wins.

**Geometric picture.** The normal equations say `X^T (Xw* - y) = 0` — the residual `Xw* - y` is **orthogonal to every column of X**. Geometrically, `Xw*` is the orthogonal projection of `y` onto the column space of X. Any other `w` gives predictions that miss orthogonally in a suboptimal way. This is why linear regression is also called "least squares projection."

## Gradient descent gets the same answer

For large D or streaming data, GD is preferable. The update rule:

$$
w \leftarrow w - \eta \cdot \nabla L(w) = w - \eta \cdot \frac{1}{N} X^\top (X w - y)
$$

Because L is **convex** (Hessian = X^T X / N is PSD), any local minimum is global. GD from any starting point converges to `w*` (the normal-equations solution) — no worry about initialization.

<RunnableCode title="Gradient descent converges to the closed-form solution" code={gdVsClosedFormCode} packages={["numpy", "matplotlib"]} captureFig={true} />

**Choosing between closed form and GD:**

- **Small D (< 10k features), plenty of RAM, no streaming:** closed form. Fastest, exact, no hyperparameters.
- **Large D, streaming, or an already-existing pipeline:** GD (or SGD with mini-batches). Same optimum, more flexible.
- **Regularization needed (L2 = ridge):** either — closed form for ridge is `w* = (XᵀX + λI)⁻¹ X^T y`, still one equation.

## Assumptions — and how their violations look

The classical statistical treatment of linear regression rests on four assumptions:

1. **Linearity.** The true relationship is `y = X w_true + ε`. If the true relationship is quadratic or oscillatory, LR is misspecified.
2. **Independent errors.** `ε_i` for different examples are independent. Violated by time series (correlated shocks) or grouped data.
3. **Homoscedasticity.** The noise variance is the same for every example. Violated when variance grows with x (e.g., higher-income individuals have more variable spending).
4. **Normality of errors.** `ε_i ~ N(0, σ²)`. Needed for classical CI / hypothesis-test theory; not needed for a *point estimate* of w.

**Coefficient estimation itself only needs (1) and (2)** — the Gauss-Markov theorem says under linearity + independent errors + finite variance, OLS is the Best Linear Unbiased Estimator (BLUE). Confidence intervals and p-values need the other assumptions.

<RunnableCode title="Residual plots diagnose assumption violations" code={assumptionsCode} packages={["numpy", "matplotlib"]} captureFig={true} />

**Fixes for common violations:**

- **Nonlinearity** (residuals show a U-shape): add polynomial features (x², x·y, etc.), or use a nonlinear model.
- **Heteroscedasticity** (residual variance grows with x): use weighted least squares, or transform the target (`log(y)` if y is always positive).
- **Autocorrelated errors** (time-series): use ARIMA-family models or add lagged features.
- **Non-normal residuals for CIs**: bootstrap the coefficient estimates.

## Interpretability of coefficients

Once you fit `ŷ = w · x + b`, each `w_i` has a specific interpretation:

$$
w_i = \text{expected change in } y \text{ per one-unit increase in } x_i, \text{ holding other features constant}
$$

This "holding other features constant" is the CETERIS PARIBUS interpretation. It's why linear regression is central to statistics: no other ML model gives such a clean per-feature effect estimate.

**Practical rules:**

- **Standardize features before comparing coefficients.** If `x_1` is in dollars and `x_2` is in years, their coefficients are on wildly different scales. Standardize (subtract mean, divide by std) so all coefficients are per-standard-deviation changes.
- **Check for multicollinearity.** If two features are strongly correlated, individual coefficient values become unstable (one can be huge positive, the other huge negative, canceling). VIF above 10 is a red flag.
- **Beware causal interpretation.** The coefficient is the association given the model + observed data. Whether X causes Y is a separate question that no regression can answer alone — but a well-designed regression can be evidence.

<RealWorldML title="Linear probing on pretrained embeddings">

A modern LLM or vision model produces **embeddings** — dense vectors that summarize an input. For evaluating whether these embeddings capture some target property, researchers use **linear probing**: freeze the encoder, extract embeddings on a labeled dataset, fit a linear regression (or logistic regression for classification) on top.

If the probe fits well (high R² for regression, high accuracy for classification), the embeddings ALREADY encode the target signal — the pretrained model has "learned it" without task-specific supervision. If the probe fits poorly, either the target is more subtle than the embedding captures, or the embedding is bad.

<RunnableCode title="Linear probing on 64-dim frozen embeddings" code={linearProbingCode} packages={["numpy"]} />

**Why linear probing is important for LLM evaluation:**

1. **Cheap.** Fitting linear regression on 64-1024 dim embeddings takes seconds. Fine-tuning the full model can take hours or days.
2. **Interpretable.** The probe weights say WHICH embedding dimensions carry the target signal. On a large labeled task, a linear probe with high R² is convincing evidence that the pretrained model captures the concept.
3. **Standard.** Papers report linear-probe results on downstream tasks (STS, Sentence-BERT eval, ImageNet linear evaluation, etc.). Beating a linear probe by a small margin with fine-tuning tells you the model's marginal capability is limited; beating it by a lot suggests real task-specific learning.

**Fun connection:** the last layer of any classifier is itself a linear probe on the previous layer's embeddings. So "linear-probe accuracy" and "final-layer accuracy" are the same thing when the earlier layers are frozen. Fine-tuning is just "linear-probing PLUS updating the encoder."

</RealWorldML>

<CodeItYourself title="Three drills — normal equations by hand, GD variants, linear probing">

1. **Fit by hand.** For the tiny dataset `X = [[1, 1], [1, 2], [1, 3]]` (bias column + one feature) and `y = [1.5, 2.5, 3.4]`, compute w* = (XᵀX)⁻¹ X^T y by hand. Verify with `np.linalg.solve`.

2. **GD sensitivity to LR.** For the synthetic problem above, sweep learning rates lr = 0.001, 0.01, 0.1, 1.0 and watch what happens. At lr = 1.0 the largest eigenvalue of the Hessian exceeds 1/lr and GD diverges; at lr = 0.001 it converges but slowly. Find the sweet spot.

3. **Linear probe on toy embeddings.** Generate synthetic "embeddings" `E = np.random.randn(500, 128)` and a target `y = E[:, :10].sum(1) + noise` — only the first 10 dimensions carry signal. Fit a linear probe on the first 250 examples, evaluate on the rest. R² should be high because the linear model can rediscover the "signal is in first 10 dims" structure automatically.

</CodeItYourself>

<Summary>

- **Model:** ŷ = w · x + b (with `x, w ∈ R^D`, `b ∈ R`). Batch: ŷ = Xw using bias trick.
- **Loss:** MSE L(w) = (1/2N) · ‖Xw - y‖². Squared for smoothness, closed-form solvability, and MLE justification under Gaussian noise.
- **Normal equations:** w* = (XᵀX)⁻¹ X^T y. Requires rank(X) = D. Compute via np.linalg.solve or np.linalg.lstsq — never inv().
- **Gradient descent** gets the same w* (loss is convex). Better when D exceeds ~10⁴ or data is streaming.
- **R² = 1 - RSS/TSS** measures fraction of variance explained. Standardize features before comparing coefficients.
- **Four classical assumptions:** linearity, independent errors, homoscedasticity, normality. Coefficient point estimates only need the first two.
- **Residual plots diagnose violations:** U-shape = nonlinearity, fan-out = heteroscedasticity.
- **Linear probing** is linear regression on top of a pretrained encoder's frozen embeddings — the standard evaluation for representation quality.

</Summary>
