export const coreMetricsCode = `# The four core regression metrics side-by-side on a common dataset.
import numpy as np

np.random.seed(0)
N = 500
y_true = 10 + 2 * np.random.randn(N)              # true values around 10 +/- 2
y_pred = y_true + 0.5 * np.random.randn(N)        # predictions with small noise

def mse(y, p):    return np.mean((p - y) ** 2)
def rmse(y, p):   return np.sqrt(np.mean((p - y) ** 2))
def mae(y, p):    return np.mean(np.abs(p - y))
def mape(y, p):   return np.mean(np.abs((p - y) / y)) * 100         # percentage
def r2(y, p):
    rss = np.sum((y - p) ** 2)
    tss = np.sum((y - y.mean()) ** 2)
    return 1 - rss / tss

print(f"MSE   = {mse(y_true, y_pred):.4f}      (squared units of y)")
print(f"RMSE  = {rmse(y_true, y_pred):.4f}     (same units as y)")
print(f"MAE   = {mae(y_true, y_pred):.4f}      (robust to outliers)")
print(f"MAPE  = {mape(y_true, y_pred):.4f}%    (percentage error)")
print(f"R^2   = {r2(y_true, y_pred):.4f}       (fraction of variance explained; 1 = perfect)")
print()
print("Rule of thumb: report both RMSE and MAE — big gap between them = outliers dragging RMSE up.")`;

np.random.seed(0)

y_good     = 2 * x + 1 + 0.5 * np.random.randn(200)          # linear + iid Gaussian noise

    return X @ w, y - X @ w

<Motivation title="Different metrics reward different behaviors — pick the metric that matches your cost">

A regression model predicts a continuous value. But "how good is the prediction" doesn't have a single answer. Consider:

- **You're forecasting inventory demand.** Missing 20 units when you predicted 100 (20% off) is worse than missing 5 units when you predicted 200 (2.5% off) — same absolute error, very different business impact. You want **percentage-based** metrics.
- **You're pricing risk.** A single outlier of ±$1M matters much more than 100 small mistakes of ±$100. You want **squared-error** metrics that magnify big mistakes.
- **You're predicting delivery time.** Being 2 hours late is much worse than being 2 hours early (customer complains vs happily surprised). You want **asymmetric** metrics.
- **You're building a warehouse restocking system.** You need a 95th-percentile prediction ("worst-case demand this week"), not the average. You want **quantile** regression.

For a GenAI engineer, choosing the right metric matters in several places:

1. **LLM eval benchmarks** (BLEU, ROUGE, BERTScore, LLM-as-judge) are all regression-like scores on a continuous quality scale.
2. **Reward-model training** for RLHF fits scalar rewards from preference labels. The loss you choose (MSE vs. Bradley-Terry log-loss vs. cross-entropy on preference pairs) shapes the reward's calibration.
3. **Latency prediction / capacity planning.** Quantile regression for "how long will this request take at the 99th percentile?" is a common ML-ops question.
4. **Perplexity for language models** is exp(mean cross-entropy per token) — a specific transformation of a regression-like loss over token probabilities.

This chapter builds the core regression metrics (MSE, RMSE, MAE, MAPE, R²), the outlier-robust alternatives (Huber, quantile), and the residual-plot diagnostics that reveal when to switch.

</Motivation>

<Intuition title="Different losses penalize errors differently; the model minimizes what you optimize">

The two most-basic regression losses:

- **MSE (Mean Squared Error):** `(1/N) · Σ (ŷ − y)²`. Squares each error. Punishes big errors much more than small ones. The estimator that minimizes MSE with a constant predictor is the **MEAN** of y.
- **MAE (Mean Absolute Error):** `(1/N) · Σ |ŷ − y|`. Uses absolute value. Punishes each error linearly, so outliers matter less. The estimator that minimizes MAE with a constant predictor is the **MEDIAN** of y.

Same data, different losses, different optimal predictions. The METRIC you optimize shapes what the model becomes.

**Rule of thumb by loss shape:**

- **MSE / RMSE:** you care about big errors more (they're squared). Sensitive to outliers.
- **MAE:** all errors matter equally per unit. Robust to outliers.
- **Huber:** MSE for small errors, MAE for large — best of both when you have some outliers you don't want to fully ignore.
- **MAPE:** scale-free (compares fractional error, not absolute). Good when scale varies wildly (predicting revenue from $10 startups to $10B companies).
- **Quantile loss (pinball):** targets a specific percentile. Good for asymmetric costs and prediction intervals.

</Intuition>

<Recall term="MSE (Mean Squared Error)">

MSE(y, ŷ) = (1/N) · Σᵢ (ŷᵢ − yᵢ)². Same units-squared as y (multiply units by 2 — e.g., "dollars²"). RMSE = √MSE puts it back into the units of y. Convex, differentiable everywhere, matches the maximum-likelihood estimator under Gaussian noise. The default regression loss.

</Recall>

## The four core metrics

**MSE (Mean Squared Error):**

$$
\mathrm{MSE} = \frac{1}{N} \sum_i (\hat{y}_i - y_i)^2
$$

Units: y-squared (usually not directly interpretable — see RMSE). Punishes big errors quadratically. Best statistic when errors are Gaussian.

**RMSE (Root Mean Squared Error):**

$$
\mathrm{RMSE} = \sqrt{\mathrm{MSE}}
$$

Same units as y. Directly interpretable ("on average, off by X y-units"). Standard for reporting.

**MAE (Mean Absolute Error):**

$$
\mathrm{MAE} = \frac{1}{N} \sum_i |\hat{y}_i - y_i|
$$

Robust to outliers (each error contributes linearly, not squared). Also in units of y.

**MAPE (Mean Absolute Percentage Error):**

$$
\mathrm{MAPE} = \frac{1}{N} \sum_i \left| \frac{\hat{y}_i - y_i}{y_i} \right| \cdot 100\%
$$

Scale-free. Compare model performance across problems with different y scales. Watch for the y_i = 0 trap.

**RMSE vs MAE comparison rule:** if RMSE >> MAE, you have outliers. If RMSE ≈ MAE, errors are roughly uniform. Report both.

## R² and adjusted R²

**R² (coefficient of determination):**

$$
R^2 = 1 - \frac{\mathrm{RSS}}{\mathrm{TSS}} = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2}
$$

Interpretation: fraction of the target's variance explained by the model. Ranges from 1 (perfect) down to 0 (baseline = predicting the mean) and can go NEGATIVE (worse than predicting the mean).

**R² for regression is like accuracy for classification** — a single number that summarizes model quality. But like accuracy, it hides details:

- Ignores the SCALE of errors (a model with R² = 0.9 could be off by 10% or 100% on average, depending on target variance).
- Sensitive to outliers (RSS is squared).
- Only makes sense on datasets with meaningful target variance.

**Adjusted R²** penalizes R² for the number of features (prevents it from artificially rising with irrelevant features):

$$
\text{adj\_}R^2 = 1 - (1 - R^2) \cdot \frac{N - 1}{N - D - 1}
$$

where D is the number of features. Standard for model comparison — the model with the higher adjusted R² is preferred over one with more features but similar R².

## Robust regression — Huber loss

When your data has outliers, MSE penalizes them harshly (squared) — the model spends most of its capacity fitting the outliers instead of the bulk of the data. MAE is more robust but has issues at zero (non-differentiable, so gradient methods use subgradients).

**Huber loss** combines the two:

$$
L_\delta(r) = \begin{cases} \frac{1}{2} r^2 & \text{if } |r| \le \delta \\ \delta \cdot (|r| - \frac{\delta}{2}) & \text{if } |r| > \delta \end{cases}
$$

where r = ŷ − y is the residual. Quadratic near zero (differentiable, well-behaved), linear far away (outliers don't blow up gradients). Parameter δ (typically 1.35 for near-Gaussian noise) controls where the transition happens.

**In sklearn:** `HuberRegressor` fits with Huber loss. Robust to ~5-10% outliers without needing to remove them.

## Quantile regression

Standard regression predicts the CONDITIONAL MEAN E[y | x]. Sometimes you want a specific quantile — e.g., "the 95th percentile of delivery time given features x" for capacity planning.

**Quantile regression** fits models to predict the q-th percentile. The loss is the **pinball loss** (asymmetric absolute error):

$$
L_q(r) = \begin{cases} q \cdot r & \text{if } r \ge 0 \\ (q - 1) \cdot r & \text{if } r < 0 \end{cases}
$$

For q = 0.5, this reduces to MAE (median regression). For q = 0.95, over-predictions are penalized 19× more than under-predictions — the model learns to sit above 95% of the data.

**Uses:**

- **Prediction intervals.** Fit two quantiles (say 5% and 95%); the interval [q_0.05, q_0.95] contains 90% of the actual y values.
- **Asymmetric costs.** Under-stocking (miss = lost sale) vs over-stocking (miss = waste). Different quantiles for each side.
- **Value at risk in finance.** 99% VaR = "what's the 1% worst-case loss?" is a quantile regression problem.

## Residual analysis

Once you've fit a model, PLOT the residuals `y - ŷ` against various things:

- **vs. predicted values ŷ:** should show scatter around zero with no pattern.
  - **U-shape:** nonlinearity in the true relationship.
  - **Fan-out (wider at extremes):** heteroscedastic noise.
- **vs. each feature x_i:** should show scatter around zero with no pattern.
  - **Linear pattern:** you're missing a term in x_i.
  - **Curve:** you need polynomial or nonlinear features of x_i.
- **Q-Q plot of residuals vs Gaussian:** should be a straight line if residuals are Gaussian.
  - **S-shape at ends:** heavy tails (outliers matter more than Gaussian assumes).
  - **Systematic curve:** residual distribution isn't Gaussian.

**Diagnostic algorithm:** always plot residuals after fitting. If they show structure, your model is misspecified. Fix by adding polynomial features (for nonlinearity), using WLS (for heteroscedasticity), or switching to a nonlinear model.

<RealWorldML title="Regression metrics in LLM evaluation">

LLM systems have several regression-flavored eval metrics:

**Perplexity (perp).** For a language model, `perp = exp(mean CE per token)`. Lower is better. It IS a regression metric — the model outputs a probability, cross-entropy compares to the one-hot label, exp makes the number more interpretable as "effective vocabulary size the model is uncertain over." A model with perplexity 2.7 says "on average, I'm uncertain over 2.7 tokens per position" — much sharper than a random model with perplexity = vocab_size.

**BLEU / ROUGE for translation and summarization.** N-gram overlap between generated and reference text. BLEU counts precision; ROUGE counts recall. Not a "true" regression — they're bounded in [0, 1] — but function similarly (higher = better). All modern LLM evals report these alongside human evaluations.

**BERTScore.** Uses BERT embeddings to measure semantic similarity between generated and reference. Continuous score, effectively a regression metric. More robust than BLEU/ROUGE for tasks where multiple valid outputs exist.

**LLM-as-judge grades.** GPT-4 rates outputs on a 1-10 scale. This is a regression score. Standard practice: report BOTH mean and standard deviation (some outputs are consistently rated, others high variance).

**Reward models for RLHF.** Reward is a scalar. Metrics: how well the reward model's outputs correlate with human preference. Kendall tau or Spearman correlation on preference pairs.

**Latency prediction for capacity planning.** "Given features (prompt length, model, complexity), predict median vs 99th percentile latency." Straight quantile regression.

**Cost prediction.** "For a workload, predict expected API cost." Standard regression, likely with MAPE as the primary metric (percentage errors matter for budgeting).

**Retrieval scores.** RAG retrieval sometimes reports "cosine similarity of retrieved chunks with query" as a diagnostic — that's a regression metric on the [-1, 1] scale.

</RealWorldML>

<CodeItYourself title="Three drills — RMSE/MAE gap, quantile-based intervals, residual detective">

1. **Detect outliers via RMSE/MAE gap.** Generate 1000 clean predictions with small noise, then inject 5 outliers (predictions off by 100×). Compute RMSE and MAE. The RMSE should be much larger; that's the outlier signal.

2. **Build a 90% prediction interval.** Fit two quantile regressions (q = 0.05 and q = 0.95) on your data. On a held-out set, verify that ~90% of the true y values fall within [q_0.05, q_0.95]. This is the empirical coverage of your interval.

3. **Residual detective.** Fit a linear regression to `y = 2·x + 0.5·x² + noise`. Plot residuals vs x. Identify the U-shape (nonlinearity). Add a quadratic feature, refit. Verify residuals are now flat.

</CodeItYourself>

<Summary>

- **MSE / RMSE:** squared error; standard for Gaussian noise; sensitive to outliers.
- **MAE:** absolute error; robust to outliers; less "smooth" for optimization.
- **MAPE:** percentage error; scale-free; watch for y = 0.
- **R²:** fraction of variance explained; use adjusted R² when comparing models with different numbers of features.
- **Huber loss:** MSE + MAE hybrid; robust to outliers while remaining differentiable.
- **Quantile regression:** predict a specific percentile via pinball loss; great for intervals and asymmetric costs.
- **Residual plots** diagnose misspecification: U-shape = missing nonlinearity; fan-out = heteroscedasticity.
- **Always report both RMSE and MAE**; big gap = outliers.
- **LLM eval metrics** (perplexity, BLEU, BERTScore) all inherit from regression metric thinking.

</Summary>
